# 官网监控架构设计

## 当前定位

已确认：

- 官网监控属于蜂巢·大圣平台辅助子系统方向。
- 本文当前仅作为官网监控规划入口和边界说明。
- 本文不代表 Monitor 仓库已启用。
- 本文不代表官网监控程序已实现、已部署或已可运行。
- 当前证据等级为 D0 / D1：已有文档入口和规划方向，但未完成源码级复核、测试运行或部署验证。

待确认：

- 是否已有独立 Monitor 仓库源码。
- 是否需要独立仓库维护，还是并入 Tools / Verify / 其他工具链。
- 首批监控目标网站、公告页面、更新频率和通知方式。
- 是否需要登录态、Cookie、代理、验证码处理。
- 是否需要接入 Verify、PCControl 或外部通知平台。
- 是否需要与游戏官网更新公告监控程序形成独立交付物。

---

## 目标

官网监控方向的长期目标是：

- 监控指定游戏官网、公告页、版本更新页或活动页。
- 发现新增公告、版本更新、维护通知、停服通知等变化。
- 记录变更时间、标题、链接、摘要和抓取状态。
- 在需要时推送到指定通知渠道。
- 为后续游戏适配、脚本更新和运营维护提供信息来源。

说明：以上是规划目标，不代表当前已实现。

---

## 非目标

- 本文不证明官网监控程序已经实现。
- 本文不证明 Monitor 仓库已经启用。
- 本文不承诺已经完成任何游戏官网抓取。
- 本文不承诺已经解决验证码、登录态、反爬、代理池问题。
- 本文不记录真实账号、密码、Cookie、Token、代理订阅地址或生产通知密钥。
- 本文不替代后续源码设计、任务调度设计、部署方案或测试报告。

## 模块边界建议

| 模块 | 职责 | 当前口径 |
|---|---|---|
| 监控目标配置 | 记录游戏官网、公告页、版本页等目标 | 规划中，未确认配置结构 |
| 抓取器 | 请求网页、获取 HTML / JSON / RSS 等内容 | 规划中，未确认技术路线 |
| 解析器 | 从页面中提取标题、时间、链接、正文摘要 | 规划中，需按游戏站点差异适配 |
| 去重与变更检测 | 判断公告是否新增、更新或删除 | 规划中，需定义 hash / diff 规则 |
| 存储层 | 保存公告记录、抓取日志和错误信息 | 规划中，未确认数据库或文件存储 |
| 调度器 | 定时执行抓取任务 | 规划中，未确认 Celery / APScheduler / cron |
| 通知器 | 发送变更提醒 | 规划中，未确认 Telegram / 企业微信 / 邮件 / 其他 |
| 管理界面 | 查看目标、日志、公告和失败任务 | 后续可选，不在本轮确认 |

## 证据等级边界

当前本文只能作为 D0 / D1 级文档入口。

| 等级 | 当前状态 |
|---|---|
| D0 | 已有草稿入口 |
| D1 | 已有规划性文档描述 |
| C1 | 未确认源码存在 |
| T1 | 未确认测试文件存在 |
| R1 | 未运行单仓测试 |
| E2 | 未完成与 Verify / PCControl / AndroidScript 联调 |
| M1 | 未完成真实部署或实际监控验证 |

没有 R1 / E2 / M1 证据时，不得写"已闭环""已上线""已部署""生产可用"。

---

## 安全边界

- 如监控目标需要登录，真实账号密码不得写入本文。
- Cookie、Session、Token、代理订阅地址、通知机器人 Token 不得写入本文。
- 只允许记录变量名、配置项名称和保管位置。
- 示例 URL 可以保留，但不得包含真实敏感参数。
- 如后续接入通知平台，通知密钥必须进入本机配置、服务器环境变量或 CI/CD Secret，不进入 Obsidian 普通 Markdown。
