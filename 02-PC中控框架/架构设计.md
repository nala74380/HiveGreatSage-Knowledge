```

       需自动适配其他分辨率（Qt DPI 缩放）

假设3：中控启动后常驻运行，定时拉取云端数据刷新界面
假设4：ADB 工具包内置于 tools/adb/platform-tools/，调用方式为 subprocess
假设5：组队模块（局域网通信）Phase 3 实现，当前仅预留接口
```

### 4.2 DPI 自动缩放（已定稿）

```python
# main.py —— 必须在 QApplication 创建之前设置
import os
os.environ["QT_AUTO_SCREEN_SCALE_FACTOR"] = "1"

from PySide6.QtWidgets import QApplication
from PySide6.QtCore import Qt

QApplication.setHighDpiScaleFactorRoundingPolicy(
    Qt.HighDpiScaleFactorRoundingPolicy.PassThrough
)
app = QApplication(sys.argv)
app.setAttribute(Qt.AA_EnableHighDpiScaling, True)
app.setAttribute(Qt.AA_UseHighDpiPixmaps, True)
```

## 五、ADB 模块（已定稿）

详见 [[ADB管理模块]]，此处记录关键集成点。

### 5.1 初始化位置

```python
# core/app.py — Application.__init__()
from core.utils.adb_manager import AdbManager
self.adb = AdbManager()
self.adb.start_server()   # 拉起 adb server（程序启动时调用一次）
```

### 5.2 工具包路径约定

```
tools/adb/platform-tools/adb.exe
```

`AdbManager` 通过 `Path(__file__).resolve().parents[2]` 定位项目根目录， 不依赖工作目录，打包后路径同样有效。

### 5.3 连接模式

|模式|Serial 格式|切换方式|
|---|---|---|
|USB|设备序列号（如 `SN·4F2A·8C1E`）|默认，USB 线连接即可|
|TCP/IP|`ip:port`（如 `192.168.1.101:5555`）|先 USB 调用 `enable_tcpip()`，再拔线|

### 5.4 激活命令序列

```bash
adb shell cp /sdcard/Download/proxy /data/local/tmp/proxy
adb shell chmod 777 /data/local/tmp/proxy
adb shell /data/local/tmp/proxy --daemon
```

任一步骤失败即中止，返回失败步骤和输出信息给 UI 层显示。

## 六、应用生命周期与启动流程

```
main.py
  │
  ├── 设置 DPI 缩放环境变量
  ├── 创建 QApplication
  │
  ├── Application.__init__()
  │     ├── 加载配置（default.yaml → local.yaml）
  │     ├── 初始化日志系统
  │     └── 初始化 AdbManager + start_server()
  │
  ├── 显示登录窗口（ui/login_window.py）
  │     ├── 读取 keyring 密码 + last_user.json 用户名（记住账号）
  │     ├── 用户输入 → Enter 或点击登录 → 调用 auth_manager.login()
  │     │     ├── 成功 → 返回 Token + 用户授权信息（含设备配额、到期时间）
  │     │     └── 失败 → 显示错误提示，留在登录窗口
  │     └── 登录成功 → 关闭登录窗口
  │
  ├── 检查热更新（update_checker）
  │     ├── 有更新 → 弹出更新对话框 → 下载 → 重启
  │     └── 无更新 → 继续
  │
  ├── 显示主窗口（ui/main_window.py）
  │     ├── 初始化四页导航（设备/组队/订单/跨服价格）
  │     ├── 初始化顶栏（账号信息 + 授权额度 + 到期时间 + 连接状态）
  │     ├── 初始化游戏定制 UI（game/dashboard_widget.py 等）
  │     └── 启动数据同步（sync_manager，后台线程）
  │
  └── 进入 Qt 事件循环 app.exec()
```

## 七、线程模型（关键设计）

```
主线程（UI 线程）
  · Qt 事件循环
  · 所有 UI 操作
  · 接收工作线程信号 → 更新 UI

同步工作线程（QThread）
  · SyncWorker：定时拉取云端设备数据
  · 信号：data_received → device_table_widget.refresh()
  · 信号：error_occurred → status_bar.show_error()

ADB 操作线程（QThread 或 QThreadPool）
  · 激活命令为阻塞操作（subprocess），不能在 UI 线程执行
  · 执行完成后通过信号通知 UI 层刷新激活状态列
  · 信号：activation_finished(serial, success, message)
```

> ⚠️ **硬规则：** 所有 subprocess / HTTP 调用必须在工作线程中执行， 结果通过 Qt Signal 传回主线程，禁止在 UI 线程中直接调用。
