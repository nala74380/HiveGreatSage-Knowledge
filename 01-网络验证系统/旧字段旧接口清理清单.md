# 旧字段旧接口清理清单

← 返回 [[项目总大纲]] | 父节点: [[01-网络验证系统/架构设计]]

## 一、清理原则

当前仍处于开发期，原则是：

```text
不兼容旧字段。
不保留旧接口。
不保留空壳文件。
不把旧命名当兼容层长期存在。
```

## 二、旧字段状态

| 对象 | 旧字段 | 当前状态 | 处理结论 |
|---|---|---|---|
| User | `user_level` | ORM 已无此字段 | 不兼容，禁止业务读取 |
| User | `max_devices` | ORM 已无此字段 | 不兼容，禁止业务读取 |
| User | `expired_at` | ORM 已无此字段 | 不兼容，禁止业务读取 |
| Token | `level` | 已改为 `authorization_level` | 不兼容旧字段 |
| Token | `project` | 已改为 `project_code` | 不兼容旧字段 |
| Agent | `level` | 旧语义易混淆 | 对外统一写 `hierarchy_depth`；业务等级写 `tier_level` |
| AgentBalance | 整表 | ORM 已删除 | 不兼容旧表 |
| BalanceTransaction | 整表 | ORM 已删除 | 不兼容旧表 |
| AuthorizationCharge | 整表 | ORM 已删除 | 不兼容旧表 |

## 三、旧接口 / 旧文件状态

| 接口或文件 | 当前状态 | 处理结论 |
|---|---|---|
| `app/routers/balance_admin.py` | 空文件残留已识别 | 删除 |
| `app/routers/balance_agent.py` | 空文件残留已识别 | 删除 |
| `scripts/setup_game_db.py` | 与 `reset_to_clean.py` 内容重复且命名危险 | 删除；如需建库脚本另建 `provision_game_db.py` |
| `DELETE /admin/api/users/{user_id}` | 旧 Admin 删除入口，绕过 user_service | 删除；统一走 `/api/users/{user_id}` 软删除 |
| `DELETE /admin/api/agents/{agent_id}` | 旧硬删除入口 | 删除；代理只允许停用，不物理删除 |
| `DELETE /admin/api/projects/{project_id}` | 旧硬删除入口 | 删除；项目只允许停用，不物理删除 |
| `/admin/api/agents/{agent_id}/balance` | 旧余额接口 | 删除；使用 `/admin/api/accounting/wallets/{agent_id}` |
| 旧 balance 账务路径 | 已迁移 | 新主线为 `accounting + prices + agents/my/balance` |

## 四、前端命名残留

| 文件 | 当前状态 | 处理策略 |
|---|---|---|
| `frontend/src/api/admin/balance.js` | 命名残留 | 后续重命名为 accounting / pricing，不做旧后端兼容 |
| `frontend/src/api/agent/balance.js` | 命名残留 | 后续重命名为 wallet，不做旧后端兼容 |
| `AgentBalance.vue` | 页面名残留 | 后续重命名为 AgentWallet 或 AgentAccounting |

## 五、禁止规则

```text
禁止新增任何读取 User.user_level / User.max_devices / User.expired_at 的代码。
禁止新增任何 level/project 旧 Token 字段兼容逻辑。
禁止恢复 balance_admin.py / balance_agent.py。
禁止通过 Admin 路由硬删除用户、代理、项目。
禁止把 setup_game_db.py 用作 reset 脚本名称。
```
