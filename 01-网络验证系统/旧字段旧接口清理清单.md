# 旧字段旧接口清理清单


## 旧字段

| 对象 | 旧字段 | 当前风险 | 清理策略 |
|---|---|---|---|
| User | user_level | 与 Authorization.user_level 冲突 | 停止业务读取，仅兼容展示，最终迁移删除 |
| User | max_devices | 与 Authorization.authorized_devices 冲突 | 前端和文档先改，后续迁移删除 |
| User | expired_at | 与 Authorization.valid_until 冲突 | 登录链路不应全局拦截，需重构 |
| Agent | level | 组织层级易与业务等级混淆 | 文档统一写 org_level / tree_level 语义 |
| AgentBalance | 整表 | 与 AccountingWallet 并存 | 迁移后删除 |
| BalanceTransaction | 整表 | 与 AccountingLedgerEntry 并存 | 迁移后删除 |

## 旧接口 / 旧路径

| 接口或文件 | 状态 | 建议 |
|---|---|---|
| `/admin/api/balance-transactions` | 旧点数流水 | 迁移到 `/admin/api/accounting/ledger` |
| `/admin/api/prices/*` | 旧项目定价 | 合并到项目详情或账务中心规则 |
| `/admin/api/agents/{agent_id}/balance` | 旧余额 | 改读 AccountingWallet |
| `/api/agents/catalog` | 兼容旧路径 | 保留观察期后删除 |
| `/admin/api/updates/{GAME_PROJECT_CODE}/{client_type}` | 旧测试路径 | 改为 project_id 路径 |
| `/admin/api/devices/{GAME_PROJECT_CODE}` | 旧测试路径 | 改为 query 参数筛选 |

## 前端残留

- UserDetail.vue：旧 user_level / max_devices / expired_at。
- DashboardView.vue：旧 user_level / expired_at。
- GuideView.vue：旧 max_devices 文案。
- BalanceTransactions.vue / ProjectPricing.vue / AgentBalance.vue：旧 balance API。

## 测试残留

- 创建用户请求体仍可能使用旧 user_level / max_devices / expired_at。
- 热更新测试仍可能使用 GAME_PROJECT_CODE 路径。
- 设备后台测试仍可能使用旧项目路径。
