# 数据库模型清单


> 主库 27 模型，游戏库 4 模型；AuthorizationFreezeRecord 已补入当前模型清单

## 当前证据边界

- 本清单基于当前 Verify main 分支 ORM 静态核对。
- 本轮未运行数据库迁移。
- 本轮未连接 PostgreSQL 检查真实物理表。
- 本轮未运行 pytest。
- "模型存在"表示 C1 级源码静态证据，不代表 R1 / E2 / M1 闭环。

## 主库模型

### 基础身份与项目 (9)
- Admin (status: active/suspended)
- Agent (hierarchy_depth 组织层级; parent_agent_id 自引用)
- User (token_version, is_deleted 软删除)
- GameProject (project_type: game/verification)
- Authorization (user_level/authorized_devices/valid_until 项目维度)
- AgentProjectAuth (source: admin_manual/request_approved/auto_approved)
- LoginLog (device_fingerprint_hash 替代原文)
- DeviceBinding (UNIQUE user+project+device; imsi Fernet 加密; imsi_hash)
- VersionRecord (partial unique index 保证每项目每客户端仅一 active)

### 项目定价 (1)
- ProjectPrice (points_per_device)

### 账务中心与授权冻结权益 (10)
- AccountingWallet (computed: available_credit, available_total)
- AccountingDocument (9 种单据类型)
- AccountingLedgerEntry (idempotency_key 防重复)
- AuthorizationChargeSnapshot (refund_status 返点追踪)
- AuthorizationFreezeRecord (authorization_freeze_record；授权停用后的剩余权益冻结记录；迁移 0023)
- AccountingReconciliationRun / Item
- AccountingAdjustmentRequest
- AccountingRiskEvent
- AgentMonthlyBill

### 代理业务画像与项目准入 (4)
- AgentBusinessProfile (tier_level 业务等级 Lv.1-4)
- AgentLevelPolicy (默认授信/下级代理/自动开通策略)
- ProjectAccessPolicy (visibility_mode + open_mode)
- AgentProjectAuthRequest (pending/approved/rejected/cancelled/auto_approved)

### 系统设置 (1)
- SystemSetting (category+key KV 结构)

### 审计 (1)
- AuditLog (actor_type/action/target_type/metadata_json 脱敏)

## 游戏库模型 (4)

- ProjectConfig (KV 项目全局配置)
- ScriptParamDef (param_type: int/float/string/bool/enum)
- UserScriptParam (user_id+param_def_id UPSERT)
- DeviceRuntime (device_id PK, JSONB game_data)

注: VersionRecord 已从游戏库迁至主库 (D014)。游戏库不再包含此表。

当前主库模型总数：27。
其中 AuthorizationFreezeRecord 已在 accounting.py、0023 迁移、查询服务、路由和测试中确认。
