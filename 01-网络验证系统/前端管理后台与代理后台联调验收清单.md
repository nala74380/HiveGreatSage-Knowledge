# Verify 前端管理后台与代理后台联调验收清单

← 返回 [[项目总大纲]] | 父节点: [[01-网络验证系统/前后端接口对照表]]

## 一、文档目的

本文用于沉淀 `HiveGreatSage-Verify` 前端管理后台与代理后台的联调验收标准。

本文重点回答：

```text
1. 当前前端已经接入了哪些后端 API？
2. 哪些页面已经存在？
3. 哪些页面只是源码接线，尚未浏览器联调？
4. 管理员端必须验收哪些页面？
5. 代理端必须验收哪些页面？
6. 点数、项目准入、热更新、设备监控等高风险页面如何验收？
7. 哪些字段名必须和后端保持一致？
8. 401 / 403 / 404 / 500 前端应该如何提示？
9. 什么条件下可以标记“前端联调通过”？
10. 联调报告如何沉淀进 Obsidian？
```

### 2.2 待确认

以下内容当前不能写成已完成：

```text
1. 前端 npm run build 是否通过。
2. 管理员登录后菜单是否能进入所有新增页面。
3. 代理登录后菜单是否能进入所有代理页面。
4. 路由守卫是否正确区分 admin / agent。
5. http 拦截器是否正确注入 Admin Token / Agent Token。
6. _skipAuthRedirect 是否会导致登录过期时不跳转或不提示。
7. 管理端点数页面是否真实请求成功。
8. 代理端余额页面是否真实请求成功。
9. 管理端项目准入策略页面是否真实保存成功。
10. 管理端项目申请审核页面是否真实批准 / 拒绝成功。
11. 代理端项目目录是否真实显示 access_status / action_type。
12. 代理端申请项目后，管理员页面是否能立刻看到 pending 申请。
13. 管理员批准后，代理端项目目录是否变为已授权。
14. 管理员拒绝后，代理端项目目录是否显示拒绝原因。
15. 浏览器控制台是否无阻塞错误。
16. 所有 401 / 403 / 404 / 500 是否有用户可理解提示。
```

---

### 2.3 当前准确状态

当前状态应标记为：

```text
前端点数与项目准入核心页面已接入源码；
但尚未完成浏览器级联调验收；
不能直接标记为“前端已稳定”。
```

风险等级：

```text
P1 高风险。
```

原因：

```text
1. 前端是管理员和代理真实操作入口。
2. 点数和项目准入都涉及商业权限。
3. 页面能编译不等于接口权限正确。
4. 页面能打开不等于业务闭环通过。
5. 字段名漂移会造成保存失败或错误展示。
```

## 四、当前 API 接口对齐清单

## 4.1 管理端点数 API

文件：

```text
frontend/src/api/admin/balance.js
```

接口清单：

| 前端函数 | 后端接口 | 用途 |
|---|---|---|
| `globalTransactions(params)` | `GET /admin/api/balance-transactions` | 管理员查看全局点数流水 |
| `getPrices(projectId)` | `GET /admin/api/prices/{project_id}` | 获取项目定价 |
| `setPrice(projectId, userLevel, data)` | `PUT /admin/api/prices/{project_id}/{user_level}` | 设置项目单价 |
| `deletePrice(projectId, userLevel)` | `DELETE /admin/api/prices/{project_id}/{user_level}` | 删除项目单价 |
| `getBalance(agentId)` | `GET /admin/api/agents/{agent_id}/balance` | 查询代理余额 |
| `recharge(agentId, data)` | `POST /admin/api/agents/{agent_id}/recharge` | 给代理充值 |
| `credit(agentId, data)` | `POST /admin/api/agents/{agent_id}/credit` | 给代理授信 |
| `freeze(agentId, data)` | `POST /admin/api/agents/{agent_id}/freeze` | 冻结授信 |
| `unfreeze(agentId, data)` | `POST /admin/api/agents/{agent_id}/unfreeze` | 解冻授信 |
| `getTransactions(agentId, params)` | `GET /admin/api/agents/{agent_id}/transactions` | 查看某代理流水 |
| `agentsFull(params)` | `GET /admin/api/agents-full` | 含余额和项目授权的代理列表 |

请求字段必须统一：

```json
{
  "amount": 100,
  "description": "备注"
}
```

项目定价请求字段必须统一：

```json
{
  "points_per_device": 100
}
```

禁止前端使用旧字段：

```text
price_points
note
```

## 5.1 管理员点数流水页面

页面：

```text
frontend/src/views/admin/BalanceTransactions.vue
```

```text
管理员查看全局点数流水，能理解充值、授信、冻结、解冻、授权扣点、删除用户返点。
```

必须验收：

```text
□ 页面能打开。
□ 页面能调用 GET /admin/api/balance-transactions。
□ 能按 tx_type 筛选。
□ 能按 agent_id 筛选。

□ consume 流水显示用户、项目、等级、授权设备数、单价。
□ refund 流水显示原扣点、已购买小时、已用小时、已用点数、返点。
□ 分页正常。
□ 后端 401 / 403 / 500 时有提示或全局处理。
```

页面：

```text
frontend/src/views/admin/ProjectAccessPolicies.vue
```

目标：

```text
管理员按项目配置代理可见性、申请规则和自动开通规则。
```

必须验收：

```text
□ 页面能打开。
□ GET /admin/api/project-access/policies 成功。
□ 列表显示 project_id。
□ 列表显示 project_name。
□ 列表显示 project_code。
□ 列表显示 project_type。
□ 列表显示 visibility_mode。
□ 列表显示 open_mode。
□ 列表显示 min_visible_agent_level。
□ 列表显示 min_apply_agent_level。
□ 列表显示 min_auto_open_agent_level。
□ 列表显示 min_available_points。
□ 列表显示 allow_apply。
□ 列表显示 allow_auto_open。
□ 列表显示 is_active。
□ 点击编辑能打开弹窗。
□ 编辑 public / level_limited / invite_only / hidden。
□ 编辑 manual_review / auto_by_level / auto_by_condition / disabled。
□ 保存时 PATCH /admin/api/project-access/policies/{project_id}。
□ 保存成功后刷新列表。
```

前端已做的校验：

```text
1. allow_auto_open=true 时，open_mode 必须为 auto_by_level 或 auto_by_condition。
2. 自动开通模式必须设置 min_auto_open_agent_level。
3. min_apply_agent_level 不能低于 min_visible_agent_level。
```

必须确认后端也有同样校验。

风险点：

```text
1. 前端校验不能替代后端校验。
2. hidden 模式后端必须强制禁止申请和自动开通。
3. invite_only 白名单管理入口当前仍待确认。
4. cooldown_hours_after_reject 是否后端真正执行仍待确认。
```

## 5.5 代理项目目录页面

页面：

```text
frontend/src/views/agent/AgentCatalog.vue
```

目标：

```text
代理查看项目目录、准入状态、项目价格、授权到期，并提交申请或自动开通。
```

必须验收：

```text
□ 页面能打开。
□ GET /api/agents/my/project-access/catalog 成功。
□ 展示项目名称。
□ 展示项目代号。
□ 展示项目类型。
□ 展示各级别点数价格。
□ 展示准入状态。
□ 展示授权到期。
□ authorized 显示“已授权”。
□ pending 显示“申请中”。
□ rejected 显示“已拒绝”并显示拒绝原因。
□ auto_open_available 显示“立即开通”。
□ apply_available 显示“申请授权”。
□ unavailable 显示“暂不可申请”。
□ 点击申请授权弹出申请说明。
□ request_reason 为空时前端阻止提交。
□ 提交后 POST /api/agents/my/project-access/requests 成功。
□ 如果返回 auto_approved，提示“项目已自动开通”。
□ 如果返回 pending，提示“申请已提交，等待管理员审核”。
□ 自动开通确认文案明确“项目开通本身不扣点”。
```

当前重要变化：

```text
该页面已不再调用旧的:
  /api/agents/my/catalog
  /api/agents/my-projects

而是使用:
  /api/agents/my/project-access/catalog
```

风险点：

```text
1. 后端 catalog 必须返回 id 或 project_id，当前前端提交 project_id 使用 row.id。
2. access_status / action_type 必须和后端枚举完全一致。
3. 自动开通时 request_reason 为空，后端必须允许该路径。
```

## 七、错误处理验收

## 7.1 401

含义：

```text
登录态失效或 Token 无效。
```

前端行为：

```text
Admin:

```

风险：

```text

必须确认这不会让页面在 401 时静默失败。
```

## 7.2 403

含义：

```text
当前身份无权访问。
```

前端行为：

```text
显示无权访问该功能。
不要跳转到错误身份登录页。
```

## 7.3 404

含义：

```text
接口未注册、路径错误或资源不存在。
```

前端行为：

```text
开发阶段可以提示“后端尚未注册接口”。
```


含义：

```text
后端内部异常。
```

前端行为：
