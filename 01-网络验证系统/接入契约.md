# 接入契约


## 当前定位

本文件描述 PCControl 与 AndroidScript 对接 Verify 的稳定契约。字段细节以 OpenAPI 快照为准。

## 当前证据边界

- 本文是 PCControl 与 AndroidScript 对接 Verify 的契约说明。
- 本文不替代 OpenAPI 快照。
- 本文不代表 PCControl / AndroidScript 当前已经完成所有字段消费。
- 本轮未重新运行三端联调。
- 涉及发布、联调、生产可用判断时，必须同时复核 Verify main、PCControl main、AndroidScript main、OpenAPI 快照和实际调用结果。

## 项目标识边界

| 标识 | 使用场景 |
|---|---|
| project_uuid | PC / Android 终端登录 Verify |
| project_id | 管理后台内部主键，热更新上传等后台接口 |
| project_code / game_project_code | 业务展示、查询、兼容接口 |

## 授权口径

- User 是账号主体。
- Authorization 是项目级授权主体。
- 用户等级、授权设备数、到期时间必须来自 Authorization。
- 响应字段可兼容 `user_level`，但来源不得再是 User.user_level。

## 设备口径

- Android 设备占用 Authorization.authorized_devices。
- PC 中控是否占用设备数：待决策，建议不占用 Android 授权名额。
- DeviceBinding 以 user + project + device 建模。

## IMSI

- `/api/device/imsi` 源码已存在。
- 当前合规边界待决策，不应默认强制上传。
- 建议 AndroidScript 增加 `enable_imsi_upload` 配置。

## 热更新

- 客户端：`/api/update/check`、`/api/update/download`。
- 管理端：`/admin/api/updates/{project_id}/{client_type}`。
- VersionRecord 主库化：源码层已统一（D014）；待三端联调验证客户端 check/download。
- 强制更新失败时，AndroidScript / PCControl 不应继续运行旧版本。

## 网络配置

- `/api/client/network-config` 用于 D 模式网络配置下发。
- PCControl / AndroidScript 是否已消费该接口：待源码复核与三端联调。
