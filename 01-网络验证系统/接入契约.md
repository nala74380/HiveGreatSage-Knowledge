## 1. 当前任务理解

本文档用于定义当前阶段网络验证系统与客户端之间的最小接入契约，重点覆盖：

1. 安卓端脚本登录接入
2. 设备身份上报
3. 编号同步规则
4. 解绑动作的服务端语义
5. 当前不纳入接入契约的字段与能力边界

**非目标：**

- 不展开 PC 中控 UI 设计
- 不展开后台页面设计
- 不展开热更新接口细节
- 不展开代理额度接口
- 不展开项目参数下发接口

## 2. 已确认信息

### 2.1 平台接入原则

**已确认：**

- 网络验证系统是中枢，客户端通过云端 API 接入。
- 安卓脚本可打包为 APK 或 LRJ：
  - APK 用于安装使用
  - LRJ 用于热更新
- 当前设备身份以“设备序列号”为主设备指纹。
- 安卓端脚本与 PC 端中控统一采用 **账号 + 密码 + 编号 + 项目UUIDv4 + 网址 + 端口** 完成首登接入。
- 首登成功后，客户端再获得会话凭证或令牌，用于后续接口访问。
- `imsi` 不属于首登必填字段，改为登录成功后的后续上传字段。

### 2.2 设备接入口径

**已确认：**

- 编号不作为设备身份
- 编号只作为展示编号
- 用户在安卓端脚本内修改编号后，登录时同步到网络验证系统
- 后台只保留当前编号，不做编号历史
- 前端“删设备”在服务端语义上定义为“解绑”

### 2.3 当前精简字段口径

**已确认：**

当前接入契约只保留：

- 首登阶段：
  - `username`
  - `password`
  - `device_alias_no`
  - `project_uuid`
  - `server_host`
  - `server_port`
  - `hardware_serial`
  - `enhanced_device_id`
- 后续上传阶段：
  - `imsi`

当前不纳入：

- qaid
- wifi_mac
- brand
- model
- manufacturer_code
- product_code
- sdk_version
- board_id

## 4. 风险点

### 4.1 禁止客户端把编号当设备身份

客户端只允许把 `device_alias_no` 当展示编号上报。  
服务端不得把它用于设备唯一性判定。

### 4.2 禁止客户端把“删除设备”理解为物理删除

当前版本中，客户端发起“删除设备”动作时，服务端统一处理为解绑，而不是删除设备本体。

### 4.3 v1 接口以“最小可用”优先

因此本文档不追求一次性覆盖所有未来能力，而是优先固化当前已确认、已可落地的契约。

## 5. 建议方案

## 5.1 首登接口（安卓端脚本 / PC 端中控共用口径）

### 5.1.1 目标

用于完成以下动作：

1. 账号密码登录校验
2. 项目定位
3. 服务地址接入
4. 设备身份识别
5. 当前编号同步
6. 获取会话凭证或令牌

### 5.1.2 首登请求建议

**接口用途：**
安卓端脚本与 PC 端中控通过账号、密码、编号、项目UUIDv4、网址、端口完成首登接入。

**请求体建议：**

```json
{
  "username": "登录账号",
  "password": "登录密码",
  "project_uuid": "项目UUIDv4",
  "server_host": "服务网址或域名",
  "server_port": 443,
  "device_alias_no": "A-001",
  "device_identity": {
    "hardware_serial": "主设备指纹",
    "enhanced_device_id": "增强设备ID",
    "runtime_mode": "root|active|accessibility|pc",
    "client_version": "1.0.0",
    "client_type": "android_script|pc_control"
  }
}
```

### 5.1.3 字段定义

#### `username`
- 含义：登录账号
- 用途：平台统一账号体系的登录名

#### `password`
- 含义：登录密码
- 用途：与账号配套的认证密码

#### `project_uuid`
- 含义：项目唯一标识
- 当前口径：UUIDv4
- 用途：标识当前客户端属于哪个项目

#### `server_host`
- 含义：服务网址或域名
- 用途：标识当前连接目标地址

#### `server_port`
- 含义：服务端口
- 用途：标识当前连接目标端口

#### `device_identity.hardware_serial`
- 含义：主设备指纹
- 当前定稿：设备序列号

#### `device_identity.enhanced_device_id`
- 含义：辅助设备标识
- 当前不参与主设备身份判定

#### `device_identity.imsi`
- 含义：辅助设备标识
- 当前不参与主设备身份判定

#### `device_identity.runtime_mode`
- 含义：运行模式
- 说明：用于识别当前客户端运行环境
- 当前建议枚举：
  - `root`
  - `active`
  - `accessibility`
  - `pc`

#### `device_identity.client_version`
- 含义：客户端版本
- 说明：当前仅用于基础兼容与排查预留

#### `device_identity.client_type`
- 含义：客户端类型
- 当前建议枚举：
  - `android_script`
  - `pc_control`

#### `device_alias_no`
- 含义：当前展示编号
- 说明：属于首登参数之一，只作展示与管理，不参与设备身份判定

## 5.2 服务端登录处理规则

### 5.2.1 处理顺序建议

1. 校验 `username` 与 `password`
2. 校验 `project_uuid`
3. 校验 `server_host` 与 `server_port`
4. 识别设备身份：
   - 以 `hardware_serial` 作为主设备指纹
5. 同步当前编号：
   - 把 `device_alias_no` 更新到 `device_identity.current_alias_no`
6. 校验授权
7. 校验设备绑定策略
8. 生成并返回会话凭证或令牌
9. 返回登录结果

### 5.2.2 设备识别规则

**当前定稿：**

- 主判定字段：`hardware_serial`
- 辅助保留字段：`enhanced_device_id`、`imsi`
- 禁止使用 `device_alias_no` 作为设备身份判定字段

---

## 5.3 编号同步规则

### 5.3.1 当前版本规则

当前版本不单独建设“编号修改接口”，采用：

> 登录时顺带同步当前编号

### 5.3.2 处理方式

当同一主设备指纹再次登录，且：

- 本次 `device_alias_no` 与数据库当前值不同

则服务端执行：

- 更新 `device_identity.current_alias_no`

### 5.3.3 当前版本不做

- 不做编号历史记录
- 不做编号冲突历史页
- 不做编号回滚逻辑

---

## 5.4 IMSI 后续上传接口

### 5.4.1 目标

用于在首登成功后，客户端补充上传 `imsi` 字段。

### 5.4.2 设计口径

**已确认：**

- `imsi` 不属于首登必填
- `imsi` 在首登成功后再上传
- 该字段作为辅助字段入库，不参与主设备身份判定

### 5.4.3 请求建议

```json
{
  "access_token": "登录成功后返回的会话令牌",
  "project_uuid": "项目UUIDv4",
  "device_identity": {
    "hardware_serial": "主设备指纹",
    "imsi": "IMSI码"
  }
}
```

### 5.4.4 服务端处理建议

1. 校验 `access_token`
2. 校验 `project_uuid`
3. 通过 `hardware_serial` 定位设备
4. 更新 `device_identity.imsi`
5. 返回上传结果

## 5.5 删除设备 / 解绑设备契约

### 5.4.1 服务端统一语义

当前版本中，前端“删除设备”动作，在服务端统一定义为：

> 解绑设备

### 5.4.2 契约要求

客户端发起删除设备动作时，服务端应：

- 不删除 `device_identity`
- 只更新 `device_binding` 为解绑状态

### 5.4.3 当前建议的数据动作

- `device_binding.status = 'unbound'`
- 写入 `unbound_at`
- 写入 `unbind_reason`
- 写入 `unbind_by_type`

### 5.4.4 当前版本不做

- 不做独立解绑历史接口
- 不做复杂审计轨迹接口
- 不做编号历史联动
