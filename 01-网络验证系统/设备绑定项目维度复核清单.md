# 设备绑定项目维度复核清单


## 当前结论

已确认：DeviceBinding 已进入 `user_id + game_project_id + device_fingerprint` 的项目维度口径。

这份文档不再是“是否迁移”的方案，而是“迁移后是否全面落地”的复核清单。

## 已确认

- DeviceBinding 模型已有 game_project_id。
- 唯一约束方向已进入 user + project + device。
- auth_service 登录绑定会使用项目维度。
- device_service 心跳会校验 user + project + device 绑定。

## 待确认

| 项 | 状态 |
|---|---|
| 旧数据是否完整回填 game_project_id | 待确认 |
| stats_service 是否彻底切到项目维度统计 | 待确认 |
| 前端设备解绑是否传项目维度 | 待确认 |
| tests/test_device 是否覆盖多项目同指纹 | 待补 |
| PC / Android 三端联调是否证明各项目额度互不影响 | 待联调 |

## 风险

如果统计、测试、文档仍停留在旧全局绑定口径，会造成：

- A 项目的设备占用误影响 B 项目。
- 后台在线数与真实项目设备不一致。
- 授权设备数扣减口径错误。
