# 实现状态对照表


## 状态分级

| 状态 | 含义 |
|---|---|
| 源码已存在 | 静态源码中已看到模型、路由、服务或页面实现 |
| 源码层已修复 | 已修改源码或测试，但尚未通过运行验证 |
| 待运行验证 | 需要实际启动、迁移、测试或构建确认 |
| 待三端联调 | 需要 Verify + PCControl + AndroidScript 联调 |
| 待生产化 | 需要部署、安全、压测、监控、回滚闭环 |
| 待修订 | 文档或代码口径已发现漂移 |

## 推荐下一步

当前最高优先级不是继续宣称“热更新已完成”，而是按以下顺序关闭验证口：

```text
1. 运行 scripts/setup_game_db.sql，解锁测试数据库前置条件。
2. 运行 tests/test_update.py。
3. 运行 tests/test_update_admin.py。
4. 管理端上传 Android / PC 更新包。
5. PCControl 调用 /api/update/check 与 /api/update/download。
6. AndroidScript 调用 /api/update/check 与 /api/update/download。
7. AndroidScript 补齐文件级 SHA-256 校验方案并实机验证安装闭环。
```

剩余重点工作：前端旧字段清理、IMSI 合规决策、部署 token 校验、Celery Worker / Beat 生产服务化。
