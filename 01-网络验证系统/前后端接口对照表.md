# Verify 前后端接口对照表

← 返回 [[项目总大纲]] | 父节点: [[01-网络验证系统/API路由清单]]

## 一、文档目的

本文用于沉淀 `HiveGreatSage-Verify` 当前 **前端 API 封装** 与 **后端 FastAPI 路由** 的对应关系。

本文不是接口设计草稿，而是基于当前源码反向整理的接口对照表。

本文重点回答：

```text
1. 前端哪个文件调用了哪个后端接口？
2. 当前接口属于 Admin Web、Agent Web、Shared，还是 Client？
3. 当前前端路径和后端路由是否一致？
4. 哪些接口已经拆分清楚？
5. 哪些接口仍然是兼容门面？
6. 哪些接口存在旧路径、旧文档、旧测试漂移风险？
7. 下一步前端联调应该按什么顺序进行？
```

### 2.2 待确认

以下内容仍需通过 `npm run build` 与浏览器联调确认：

```text
1. 所有 import 路径是否正确。
2. 所有页面是否已切换到新拆分 API。
3. 兼容门面是否仍被旧页面引用。
4. Axios baseURL 是否正确。
5. Admin Token / Agent Token / User Token 是否正确存取。
6. 401 / 403 拦截器是否按身份正确处理。
7. 前端是否仍调用旧热更新接口。
8. 前端是否仍调用旧设备接口。
9. 前端是否仍调用旧用户删除接口。
10. 前端是否仍调用已废弃或待废弃路径。
```

### 2.3 状态标记

```text
✓ 已对齐:
  当前前端调用路径与当前后端路由一致。

□ 待联调:
  前后端源码看起来一致，但未在浏览器实际验证。

⚠️ 待确认:
  当前有兼容门面、旧路径、旧测试或路由版本漂移风险。

✗ 不建议继续使用:
  旧路径或职责不清路径，后续应迁移。
```

## 三、接口分层规则

### 3.1 Admin Web

管理员后台使用：

```text
frontend/src/api/admin/*
```

典型前缀：

```text
/admin/api/*
/api/users/*      # 管理员与代理共用时例外
/api/agents/*     # 代理管理与代理共用时例外
```

Admin Web 的身份：

```text
Admin Token
```

### 3.2 Agent Web

代理后台使用：

```text
frontend/src/api/agent/*
frontend/src/api/shared/*
```

典型前缀：

```text
/api/agents/my/*
/api/agents/my/project-access/*
/api/users/*
```

Agent Web 的身份：

```text
Agent Token
```

### 3.3 Shared

共用接口放在：

```text
frontend/src/api/shared/*
```

当前核心共用模块：

```text
shared/user.js
```

原因：

```text
/api/users/* 当前同时支持 Admin Token 与 Agent Token。
权限边界由后端 service 强制。
```

### 3.4 Client

PC 中控 / 安卓脚本用户态接口放在：

```text
frontend/src/api/client/*
```

典型前缀：

```text
/api/device/*
/api/params/*
/api/update/*
/api/auth/*
```

Client 身份：

```text
User Token
```

### 3.5 兼容门面

兼容门面用于旧页面过渡：

```text
frontend/src/api/user.js
frontend/src/api/project.js
frontend/src/api/device.js
```

长期建议：

```text
1. 新页面不要再直接依赖兼容门面。
2. 新页面应按身份导入 admin / agent / shared / client。
3. 兼容门面只保留到所有旧页面迁移完。
```

### 5.1.2 风险点

```text
1. 点数接口属于财务敏感接口，不能只做页面联调。
2. recharge / credit / freeze / unfreeze 必须有后端审计。
3. setPrice 会影响后续授权扣点，必须记录价格变更。
4. 前端只能验证页面调用成功，不能证明财务一致性。
```

## 十五、风险清单

### R001 UploadVersionForm 绕过统一 http 实例

当前：

```text
UploadVersionForm.vue 直接 import axios。
```

风险：

```text
1. baseURL 可能与 http 实例不一致。
2. Token 注入逻辑重复。
3. 401 处理逻辑不统一。
4. 生产部署子路径时可能出问题。
```

建议：

```text
新增 adminUpdateApi.uploadVersion。
组件只负责表单和进度。
```

---

### R002 兼容门面长期存在导致边界不清

涉及：

```text
src/api/user.js
src/api/project.js
src/api/device.js
```

风险：

```text
1. 新页面继续误用旧门面。
2. Admin / Agent / Client 身份边界模糊。
3. 路由迁移后旧门面隐藏问题。
```

建议：

```text
1. 保留兼容门面，但标记“旧页面兼容”。
2. 新页面禁止引入兼容门面。
3. 每次前端新增页面必须明确身份层。
```

---

### R003 Admin / Agent 共用用户接口需要后端权限测试

涉及：

```text
/api/users/*
```

风险：

```text
前端路径一致，但权限语义不同。
```

建议：

```text
补 Admin / Agent 双身份用户管理测试。
```

---

### R004 项目目录接口语义重叠

当前存在：

```text
/api/agents/my/catalog
/api/agents/my-projects
/api/agents/my/project-access/catalog
```

风险：

```text
1. 项目价格目录。
2. 已授权项目。
3. 项目准入目录。
```

三者容易混淆。

建议命名：

```text
项目准入目录:
  /api/agents/my/project-access/catalog

项目价格目录:
  /api/agents/my/catalog

已授权项目:
  /api/agents/my-projects
```

---

### R005 热更新 project_id / game_project_code 路径参数漂移

风险：

```text
前端上传使用 projectId。
旧测试可能使用 game_project_code。
```

建议：

```text
统一 V2 热更新管理接口使用 project_id。
旧测试同步更新。
```
