## 三、已发现重点问题

### 3.1 `balance.js` 中 `catalog()` 路径与后端双挂载有关

前端调用：

```javascript
http.get('/api/agents/catalog')
```

后端当前通过 `balance.router` 挂载到 `/api/agents` 后，可以形成 `/api/agents/catalog`。  
但同一个 router 也被挂到 `/admin/api`，会额外形成 `/admin/api/catalog` 等非前端显式使用路径。

**建议方案：** 将余额/价格路由拆分为 Admin 与 Agent 两组，避免双挂载造成隐式 API 面。

### 3.2 `device.js` 仍保留 User Token 设备接口口径

前端 `device.js` 注释中说明 `/api/device/list` 与 `/api/device/data` 只接受 User Token。  
但管理后台实际已经有 `/admin/api/devices/` 和 `/admin/api/devices/{game_project_code}`。

**建议方案：**

- 管理后台设备页优先调用 `device_admin` 路由。
- `/api/device/*` 保留给 PC 中控/安卓脚本用户态。
- 在前台操作手册中明确“用户设备接口”和“后台设备监控接口”的区别。

### 3.3 路由文件与前端 API 文件不是一一对应

例如：

- `frontend/src/api/user.js` 同时调用 `/api/users/*` 与 `/admin/api/users/*/devices`
- `frontend/src/api/agent.js` 同时调用 `/api/agents/*` 与 `/admin/api/dashboard`
- `frontend/src/api/balance.js` 同时包含价格、余额、流水、代理自查

**建议方案：** 前端 API 层后续也可按调用方拆分：

```text
api/admin/*
api/agent/*
api/client/*
```
