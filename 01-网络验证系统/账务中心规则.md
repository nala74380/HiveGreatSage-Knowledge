# 账务中心规则


## 当前结论

已确认：新账务中心已落地，`accounting_*` 模型与 `/admin/api/accounting/*` 是后续账务主线。

## 新主线

- AccountingWallet：代理钱包余额快照。
- AccountingDocument：业务单据。
- AccountingLedgerEntry：不可变账本明细。
- AuthorizationChargeSnapshot：授权扣点快照。
- AccountingReconciliationRun / Item：对账运行和差异项。
- AccountingAdjustmentRequest：调账申请。
- AccountingRiskEvent：风险事件。
- AgentMonthlyBill：代理月账单。

## 旧兼容层

- AgentBalance
- BalanceTransaction
- AuthorizationCharge
- balance_service.py
- balance_admin.py
- balance_agent.py
- frontend admin/agent/balance.js

这些只能作为兼容层存在，不允许新增业务继续依赖。

## 已确认风险

1. project_access_service 自动开通可用点数可能仍读取旧 AgentBalance。
2. 前端旧 BalanceTransactions / ProjectPricing / AgentBalance 页面仍残留旧 API。
3. 旧 balance_transaction 未完整迁移为 accounting_ledger_entry，需要基线初始化和对账说明。
4. 钱包状态、风险状态、幂等键、调账/冲正入口仍需补生产闭环。

## 建议迁移顺序

1. 项目准入自动开通改读 AccountingWallet。
2. user_service 扣点 / 返点直接导入 accounting_service。
3. 前端旧 balance API 全部切 accountingApi。
4. 旧表只读观察一个版本周期。
5. 完成对账后再删除旧表和旧服务。
