# 旧点数系统兼容层说明

← 返回 [[项目总大纲]] | 父节点: [[01-网络验证系统/账务中心规则]]

## 一、结论

当前开发期不保留旧点数系统兼容层。

旧系统只作为历史迁移说明存在，不作为运行时兼容接口存在。

## 二、旧系统对象

| 旧对象 | 当前状态 |
|---|---|
| `AgentBalance` | 已退出 ORM 主线 |
| `BalanceTransaction` | 已退出 ORM 主线 |
| `AuthorizationCharge` | 已退出 ORM 主线 |
| `balance_admin.py` | 空文件残留，需删除 |
| `balance_agent.py` | 空文件残留，需删除 |

## 三、当前账务主线

| 当前对象 | 用途 |
|---|---|
| `AccountingWallet` | 代理钱包 |
| `AccountingDocument` | 账务单据 |
| `AccountingLedgerEntry` | 账本流水 |
| `AuthorizationChargeSnapshot` | 授权扣点快照 |
| `AccountingReconciliationRun` | 对账批次 |
| `AccountingAdjustmentRequest` | 调账申请 |
| `AccountingRiskEvent` | 风控事件 |

## 四、当前接口主线

```text
/admin/api/accounting/*
/admin/api/prices/*
/api/agents/my/balance
/api/agents/my/transactions
```

## 五、禁止规则

```text
禁止恢复旧 balance_admin.py / balance_agent.py。
禁止新增 /admin/api/agents/{agent_id}/balance 旧接口。
禁止把旧表作为新接口的数据来源。
禁止在 Knowledge 中继续写“旧点数兼容层已保留”。
```
