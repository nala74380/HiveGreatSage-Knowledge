# 代理管理方案

> 最后更新: 2026-05-10（审查 + 范围管理补充）

## 一、核心概念

- `Agent.hierarchy_depth`：代理组织层级 / 代理树深度（数据库列名 `level`，ORM 属性 `hierarchy_depth`）。
- `AgentBusinessProfile.tier_level`：代理业务等级 Lv.1 - Lv.4。

## 二、组织架构

多级代理树通过 `parent_agent_id` 自引用 + `WITH RECURSIVE` CTE 查询：
- `app/core/utils.py:get_agent_scope_ids()` — 获取子树内所有代理 ID
- `app/services/agent_service.py:get_agent_subtree()` — 构建嵌套树形结构

代理创建规则：
- `parent_agent_id=None` → 顶级代理 (hierarchy_depth=1)，由管理员创建
- 指定 parent → hierarchy_depth = parent.hierarchy_depth + 1

## 三、代理端范围管理（14 端点）

`app/routers/agent_scope_management.py` (1225行) — 代理只能操作自己子树内的下级代理。

| 端点 | 用途 | 权限校验 |
|------|------|---------|
| `GET /scope/level-policies` | 获取下级可设置等级策略 | `get_current_agent` |
| `GET /scope/{id}/business-profile` | 查看下级业务画像 | `_target_agent_or_404`（子树校验） |
| `PATCH /scope/{id}/business-profile` | 更新下级业务画像 | 子树 + 等级只能低于自身一级 |
| `GET /scope/{id}/project-auths` | 查看下级项目授权 | 子树校验 |
| `POST /scope/{id}/project-auths` | 给下级开通项目 | 子树 + 父级须已有该项目授权 + 到期时间不超父级 |
| `PATCH /scope/{id}/project-auths/{aid}` | 更新下级项目授权 | 子树 + 父级项目约束 |
| `DELETE /scope/{id}/project-auths/{aid}` | 停用下级项目授权 | 子树校验 |
| `GET /scope/{id}/balance` | 查看下级余额 | 子树校验 |
| `GET /scope/{id}/transactions` | 查看下级流水 | 子树校验 |
| `POST /scope/{id}/recharge` | 划拨充值点数给下级 | 子树 + 上级 charged 余额充足 + `for_update` 行锁 |
| `POST /scope/{id}/credit` | 划拨授信点数给下级 | 子树 + 不超过下级 max_credit_limit + 行锁 |
| `POST /scope/{id}/freeze` | 冻结下级授信 | 子树 + 可用授信充足 + 行锁 |
| `POST /scope/{id}/unfreeze` | 解冻下级授信 | 子树 + 已冻结额充足 + 行锁 |
| `POST /scope/{id}/password` | 重置下级密码 | 子树 + 支持自动生成或手动输入 |

所有写操作均写入 audit_log，操作者 `actor_type=agent`。

## 四、代理创建下级能力检查

创建下级代理时需逐项校验（`agents.py:create_agent_in_scope_endpoint`）：
1. `can_create_sub_agents` — 业务画像/策略是否允许
2. `max_sub_agents` — 现有下级数是否已达上限
3. `child_tier_level = parent_tier_level - 1` — 下级等级必须低一级
4. `child_tier_level >= 1` — 最低 Lv.1

## 五、代理管理路由对比

| 能力 | Admin 端点 | Agent 端点 |
|------|-----------|-----------|
| 创建代理 | `POST /api/agents/` | `POST /api/agents/scope` |
| 查看列表 | `GET /api/agents/` | `GET /api/agents/scope/list` |
| 查看详情 | `GET /api/agents/{id}` | `GET /api/agents/scope/{id}` |
| 查看子树 | `GET /api/agents/{id}/subtree` | `GET /api/agents/scope/{id}/subtree` |
| 更新状态 | `PATCH /api/agents/{id}` | `PATCH /api/agents/scope/{id}` |
| 删除 | `DELETE /api/agents/{id}` (仅开发环境) | — |
