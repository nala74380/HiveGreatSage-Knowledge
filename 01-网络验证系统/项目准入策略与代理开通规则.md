项目准入系统属于：

```text
P1 高风险模块。
```

原因：

```text
1. 它决定代理能否售卖某个项目。
2. 它影响第二游戏、第三游戏接入后的商业边界。

5. 一旦误拦截，会导致合法代理无法售卖项目。
```

## 三、术语定义

## 3.1 项目准入

项目准入表示：

```text
某个代理是否有资格看到、申请、自动开通、售卖某个项目。
```

项目准入不是用户授权。

区别：

```text
代理项目准入:
  代理能不能卖这个项目。

用户项目授权:
  用户能不能登录和使用这个项目。
```

---

## 3.2 代理业务等级 tier_level

字段位置：

```text
agent_business_profile.tier_level
```

含义：

```text
代理业务等级，范围 Lv.1 - Lv.4。
```

用途：

```text
1. 判断项目是否可见。
2. 判断项目是否可申请。
3. 判断项目是否可自动开通。
4. 判断是否满足部分项目准入门槛。
```

### 重要边界

```text
Agent.hierarchy_depth:
  代理组织层级 / 代理树深度。

AgentBusinessProfile.tier_level:
  代理业务等级 Lv.1 - Lv.4。
```

禁止混用。

表名：

```text
agent_project_auth
```

职责：

```text
代理当前实际拥有的项目开通结果。
```

## 四、数据模型边界

## 4.1 AgentLevelPolicy

表名：

```text
agent_level_policy
```

职责：

```text
定义代理业务等级的默认策略。
```

当前迁移脚本中的默认等级：

```text
Lv.1 新手代理:
  默认授信 200，最高 300，不允许自动开通。

Lv.2 标准代理:
  默认授信 1000，最高 1500，可自动开通基础项目。

Lv.3 核心代理:
  默认授信 4000，最高 5000，可自动开通基础/中级项目。

Lv.4 渠道代理:
  默认授信 10000，最高 15000，可自动开通大部分非隐藏项目。
```

注意：

```text
AgentLevelPolicy.level 是业务等级。
不是 Agent.hierarchy_depth 组织树深度。
```

---

## 4.2 ProjectAccessPolicy

表名：

```text
project_access_policy
```

职责：

```text
保存某个项目的准入策略。
```

核心字段：

```text
project_id
visibility_mode
open_mode
min_visible_agent_level
min_apply_agent_level
min_auto_open_agent_level
min_available_points
allow_apply
allow_auto_open
require_request_reason
cooldown_hours_after_reject
is_active
```

默认策略：

```text
visibility_mode = public
open_mode = manual_review
min_visible_agent_level = 1
min_apply_agent_level = 1
min_auto_open_agent_level = null
min_available_points = 0
allow_apply = true
allow_auto_open = false
require_request_reason = true
cooldown_hours_after_reject = 24
is_active = true
```

### 表级约束

```text
visibility_mode IN ('public', 'level_limited', 'invite_only', 'hidden')

open_mode IN ('manual_review', 'auto_by_level', 'auto_by_condition', 'disabled')

min_visible_agent_level BETWEEN 1 AND 4

min_apply_agent_level BETWEEN 1 AND 4

min_auto_open_agent_level IS NULL OR BETWEEN 1 AND 4

min_available_points >= 0
```

## 五、策略字段说明

## 5.1 visibility_mode

字段：

```text
project_access_policy.visibility_mode
```

可选值：

```text
public
level_limited
invite_only
hidden
```

### public

含义：

```text
公开可见。
```

规则：

```text
代理只要登录代理后台，就可以在项目目录中看到该项目。
```

但是否能申请或自动开通，还要看：

```text
open_mode
allow_apply
allow_auto_open
min_apply_agent_level
min_auto_open_agent_level
```

---

### level_limited

含义：

```text
按代理业务等级限制可见。
```

规则：

```text
只有 tier_level >= min_visible_agent_level 的代理可见。
```

示例：

```text
min_visible_agent_level = 3

Lv.1:
  不可见。

Lv.2:
  不可见。

Lv.3:
  可见。

Lv.4:
  可见。
```

---

### invite_only

含义：

```text
指定代理可见。
```

规则：

```text
只有存在 agent_project_access_invite 且 is_active=true 的代理可见。
```

适合：

```text
1. 内测项目。
2. 大客户定制项目。
3. 不公开售卖项目。
4. 渠道专属项目。
```

---

### hidden

含义：

```text
隐藏项目。
```

规则：

```text
普通代理目录不展示。
即使代理等级足够，也不可见。
```

源码规则：

```text
hidden 项目会强制:
  allow_apply = false
  allow_auto_open = false
  open_mode = disabled
  min_auto_open_agent_level = null
```

用途：

```text
1. 未上线项目。
2. 内部测试项目。
3. 已下架项目。
4. 不允许代理侧看到的项目。
```

---

## 5.2 open_mode

字段：

```text
project_access_policy.open_mode
```

可选值：

```text
manual_review
auto_by_level
auto_by_condition
disabled
```

---

### manual_review

含义：

```text
必须人工审核。
```

规则：

```text
代理提交申请后，状态为 pending。
管理员批准后，才写入 agent_project_auth。
管理员拒绝后，状态为 rejected。
```

适合：

```text
1. 高价值项目。
2. 新代理不稳定。
3. 需要人工判断代理资质的项目。
```

---

### auto_by_level

含义：

```text
按代理业务等级自动开通。
```

规则：

```text
allow_auto_open = true
min_auto_open_agent_level 必须有值
tier_level >= min_auto_open_agent_level
```

满足后：

```text
1. 创建 agent_project_auth_request。
2. status = auto_approved。
3. 写 auto_approve_reason。
4. 写 agent_project_auth。
5. agent_project_auth.source = auto_approved。
```

注意：

```text
自动开通也必须写申请审计记录。
```

---

### auto_by_condition

含义：

```text
按代理业务等级 + 可用点数自动开通。
```

规则：

```text
allow_auto_open = true
min_auto_open_agent_level 必须有值
tier_level >= min_auto_open_agent_level
available_points >= min_available_points
```

其中：

```text
available_points = charged_points + credit_points - frozen_credit
```

适合：

```text
1. 需要代理有一定余额/授信后才自动开通。
2. 防止空余额代理批量开项目。
3. 中高风险项目。
```

---

### disabled

含义：

```text
禁止申请。
```

规则：

```text
代理不可申请。
不可自动开通。
```

用途：

```text
1. 暂停新代理开通。
2. 项目维护中。
3. 项目下架前过渡。
```

## 6.2 目录构造流程

当前服务层流程：

```text
1. 获取当前代理业务等级 tier_level。
2. 查询所有 active 项目。
3. 加载项目准入策略 policy。
4. 加载项目价格 price。
5. 加载当前代理已授权项目 agent_project_auth。
6. 加载当前代理最近申请记录 agent_project_auth_request。
7. 加载当前代理 invite_only 邀请。
8. 计算代理可用点数 available_points。
9. 对每个项目判断是否可见。
10. 对可见项目计算 access_status 和 action_type。
11. 返回目录项。
```

## 6.3 access_status

Schema 中定义的说明：

```text
authorized
pending
rejected
auto_open_available
apply_available
unavailable
```

### authorized

含义：

```text
代理已经拥有该项目授权。
```

判断：

```text
agent_project_auth 存在
status = active
valid_until 为空或未过期
```

前端动作：

```text
无需申请。
可显示“已授权”。
```

---

### pending

含义：

```text
代理已经提交申请，等待管理员审核。
```

## 7.2 提交申请前置校验

当前服务层已确认包含：

```text
1. 项目必须存在且 active。
2. 若已拥有 active agent_project_auth，则拒绝重复申请。
3. 若已有 pending 申请，则返回 409。
4. 项目必须对该代理可见。
5. 如果满足自动开通条件，则进入自动开通。
6. 如果不满足自动开通，但允许申请，则创建 pending 申请。
7. 如果 policy.require_request_reason = true，则 request_reason 必填。
```

---

## 7.3 自动开通流程

触发条件：

```text
_can_auto_open(...) = true
```

满足条件：

```text
1. policy.is_active = true。
2. policy.allow_auto_open = true。
3. policy.open_mode ∈ {'auto_by_level', 'auto_by_condition'}。
4. policy.min_auto_open_agent_level 不为空。
5. tier_level >= min_auto_open_agent_level。
6. 如果 open_mode = auto_by_condition:
   available_points >= min_available_points。
```

自动开通写入：

```text
agent_project_auth_request:
  status = auto_approved
  reviewed_at = now
  auto_approve_reason = 自动开通原因

agent_project_auth:
  status = active
  source = auto_approved
  request_id = req.id
  granted_by_admin_id = null
  granted_reason = auto_approve_reason
  valid_until = null
```

### 重要规则

```text
自动开通本身不扣点。
代理给用户授权项目时才扣点。
```

## 13.1 策略可见性测试

public：

```text
□ Lv.4 可见。
```

```text
□ 兄弟代理不可见。
```

hidden：

```text

□ 即使 tier_level=4 也不可见。
□ 策略保存后 allow_apply=false。
□ 策略保存后 allow_auto_open=false。
□ 策略保存后 open_mode=disabled。
```

## 13.2 申请能力测试

manual_review：

```text
□ allow_apply=true。
□ tier_level >= min_apply_agent_level。
□ 提交后 status=pending。
□ 不写 agent_project_auth。
```

allow_apply=false：

```text
□ 项目可见但不可申请。
□ POST /requests 返回 403。
```

disabled：

```text
□ 项目可见但不可申请。
□ POST /requests 返回 403。
```

```text

```


```text

□ 再次申请返回 400。
```

## 13.3 自动开通测试

auto_by_level：

```text
□ allow_auto_open=true。
□ open_mode=auto_by_level。
□ min_auto_open_agent_level=2。
□ Lv.1 不能自动开通。
□ Lv.2 自动开通。
□ Lv.3 自动开通。
□ Lv.4 自动开通。

□ 自动开通写 agent_project_auth.source=auto_approved。
```

```text
□ allow_auto_open=true 但 open_mode=manual_review，保存失败。
□ open_mode=auto_by_level 但 min_auto_open_agent_level=null，保存失败。
```
