# 服务层业务规则

← 返回 [[项目总大纲]] | 父节点: [[01-网络验证系统/架构设计]]

## 一、边界原则

Router 只负责：

```text
路由声明
请求参数接收
Depends 鉴权
调用 service
返回 response_model
```

Router 禁止负责：

```text
复杂 SQL 查询
账务计算
扣点 / 返点
权限范围递归
文件存储细节
Redis 缓存策略
多步骤业务事务
调用 service 中以下划线开头的私有函数
```

## 二、当前业务真相源

| 领域 | 当前服务层 |
|---|---|
| 认证登录 / Refresh | `auth_service.py` |
| 用户与授权 | `user_service.py` |
| 代理管理 | `agent_service.py` |
| 代理业务画像 | `agent_profile_service.py` |
| 项目管理 | `project_service.py` |
| 项目准入 | `project_access_service.py` |
| 账务 | `accounting_service.py`, `accounting_query_service.py` |
| 设备终端 | `device_service.py` |
| 后台设备 | `device_admin_service.py` |
| 参数 | `params_service.py` |
| 热更新客户端 | `update_service.py` |
| 热更新管理 | 待下沉到 `update_service.publish_version_package()` |
| 统计 | `stats_service.py` |
| 系统设置 | `system_setting_service.py` |

## 三、当前待重构项

```text
users.upgrade_preview_endpoint → user_service.preview_authorization_upgrade（已开始处理）
update_admin.upload_version_endpoint → update_service.publish_version_package
device_admin.list_all_devices → device_admin_service.get_global_device_list
agents.get_my_profile → agent_profile_service.get_agent_dashboard_profile
auth.get_me → auth_service.get_current_user_project_summary
admin.py 旧删除接口 → 删除或迁移到对应 service
```

## 四、禁止规则

```text
禁止 router import service 私有函数。
禁止在 router 中直接拼账务扣点规则。
禁止在 router 中直接物理删除代理、项目、用户。
禁止新增无 response_model 的核心商业接口。
```
