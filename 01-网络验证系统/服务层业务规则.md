# 服务层业务规则


## 当前结论

已确认：旧日期版服务层文档以 balance_service 为主，已落后于当前 accounting 主线。本文件按当前源码静态精读重建服务边界。

## 认证与授权

- `auth_service.py`：登录、刷新、登出、Token 校验、设备绑定入口。
- 正确授权口径：Authorization.user_level / authorized_devices / valid_until。
- 风险：login 仍检查 User.expired_at；refresh 仍可能使用 User.user_level。

## 用户服务

- `user_service.py`：User 是账号主体，Authorization 是项目级授权主体。
- 删除用户为软删除，涉及授权停用、设备状态、返点逻辑。
- 风险：仍通过旧 balance_service 导出名调用扣点 / 返点函数。

## 设备服务

- `device_service.py`：客户端设备绑定、心跳、IMSI、运行数据查询。
- `device_admin_service.py`：后台设备监控。
- 设备绑定口径：user + project + device。
- 风险：IMSI 明文保存和展示；统计服务仍有旧口径残留。

## 账务服务

- `accounting_service.py`：新账务中心写入主线。
- `accounting_query_service.py`：账务查询。
- `accounting_reconciliation_service.py`：对账和基线初始化。
- `balance_service.py`：旧点数兼容层，不应作为新增功能入口。

## 项目准入

- `project_access_service.py`：项目目录、申请、审批、自动开通。
- 正确等级口径：AgentBusinessProfile.tier_level。
- 风险：自动开通可用点数查询仍可能读取 AgentBalance。

## 热更新

- `update_admin.py` / 后台上传：主库项目维度路径。
- `update_service.py` / 客户端检查：仍有游戏库 VersionRecord 口径风险。

## 系统设置

- `system_setting_service.py`：网络部署模式、D 模式、relay/direct 策略。
- 风险：PCControl / AndroidScript 是否消费 `/api/client/network-config` 待确认。
