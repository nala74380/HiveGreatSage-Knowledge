## 一、记录目的

本文用于记录 Verify 风险整改的真实提交状态。

原则：

```text
已提交：GitHub main 上已经能看到对应源码或文档变化。
已确认：通过 GitHub 文件复核或接口返回确认。
待提交：已经确定整改方向，但尚未提交到 GitHub。
受阻：整改方向明确，但当前工具或环境阻止提交。
待验证：源码已改，但尚未运行 pytest、前端 build、迁移或联调。
```

本文不把“计划整改”写成“已经完成”。
