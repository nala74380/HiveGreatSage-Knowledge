# HiveGreatSage-Verify 前台操作手册

> **前台地址：** http://127.0.0.1:5173/
> **前提：** 已按 [[开发启动流程]] 启动所有服务（WSL 的 PostgreSQL + Redis，PowerShell 的后端 + 前端）。
> **原则：** 所有账号/项目创建均在前台页面操作，不直接写数据库。

---

## 操作流程总览

```
① 管理员登录
    ↓
② 创建游戏项目库（脚本）+ 注册游戏项目（API）
    ↓
③ 创建代理账号
④ 给代理授权游戏项目（代理才能为此项目创建用户）
    ↓
⑤ 代理登录
⑥ 代理创建用户账号
⑦ 代理为用户添加游戏授权
    ↓
⑧ 用户通过 PC 中控 / 安卓脚本登录
    ↓
⑨ 验证：登录日志 / 设备绑定 / 心跳上报
```

---

## 第一步：管理员登录

**接口：** `POST /admin/api/auth/login`

```json
{
  "username": "admin",
  "password": "Admin@2026!"
}
```

**返回：**
```json
{
  "access_token": "eyJ...",
  "token_type":   "bearer",
  "expires_in":   28800,
  "admin_id":     1,
  "username":     "admin"
}
```

> ⚠️ **复制 `access_token`，后续所有管理员操作需要用它。**
> 在 Swagger 右上角点「Authorize」，填入 `Bearer eyJ...`（带 Bearer 前缀）。

---

## 第二步：创建游戏项目（两步）

### 2.1 建游戏项目数据库（命令行，一次性）

```powershell
# 如果 hive_yeya 库还不存在
python scripts/setup_real_env.py
# 或单独建库：
python scripts/setup_game_db.py --code yeya --name "椰芽"
```

**输出应显示：**
```
✅ 数据库 hive_yeya 创建成功
✅ 游戏库 hive_yeya 建表完成
```

### 2.2 在 API 注册游戏项目

> 注意：如果 `setup_real_env.py` 已经注册过，这步跳过（API 会返回 409 Conflict）。

**接口：** `POST /admin/api/projects/`

```json
{
  "code_name":    "yeya",
  "display_name": "椰芽",
  "db_name":      "hive_yeya",
  "project_type": "game"
}
```

**返回（记录 project_uuid 和 id）：**
```json
{
  "id":           1,
  "project_uuid": "07238db5-129a-4408-b82a-e025be4652a1",
  "code_name":    "yeya",
  "display_name": "椰芽",
  "db_name":      "hive_yeya",
  "is_active":    true
}
```

> ⚠️ **`project_uuid` 必须与 PCControl `config/local.yaml` 和安卓 `config.lua` 中的 UUID 完全一致。**
> 当前配置值：`07238db5-129a-4408-b82a-e025be4652a1`

## 第三步：创建代理账号

**接口：** `POST /api/agents/`
**Header：** `Authorization: Bearer <admin_token>`

```json
{
  "username":     "agent_001",
  "password":     "Agent@2026!",
  "max_users":    100,
  "commission_rate": null,
  "parent_agent_id": null
}
```

**返回（记录 agent.id）：**
```json
{
  "id":               1,
  "username":         "agent_001",
  "level":            1,
  "parent_agent_id":  null,
  "max_users":        100,
  "status":           "active"
}
```

---

## 第四步：给代理授权游戏项目

> 代理只能销售/分配已被授权的项目给用户。不授权则代理无法为此项目创建用户。

**接口：** `POST /admin/api/agents/{agent_id}/project-auths/`
**Header：** `Authorization: Bearer <admin_token>`

```json
{
  "project_id":  1,
  "valid_until": null
}
```

**返回：**
```json
{
  "id":         1,
  "agent_id":   1,
  "project_id": 1,
  "status":     "active",
  "valid_until": null
}
```

---

## 第五步：代理登录

**接口：** `POST /api/agents/auth/login`

```json
{
  "username": "agent_001",
  "password": "Agent@2026!"
}
```

**返回（记录 agent_token）：**
```json
{
  "access_token": "eyJ...",
  "expires_in":   28800,
  "agent_id":     1,
  "username":     "agent_001",
  "level":        1
}
```

> 在 Swagger 右上角更新 Authorize 为代理 Token（或新开一个浏览器标签）。

---

## 第六步：代理创建用户账号

**接口：** `POST /api/users/`
**Header：** `Authorization: Bearer <agent_token>`

```json
{
  "username":    "user_001",
  "password":    "User@2026!",
  "user_level":  "normal",
  "max_devices": 500,
  "expired_at":  null
}
```

**返回（记录 user.id）：**
```json
{
  "id":            1,
  "username":      "user_001",
  "user_level":    "normal",
  "status":        "active",
  "max_devices":   500,
  "expired_at":    null
}
```

---

## 第七步：为用户添加游戏授权

**接口：** `POST /api/users/{user_id}/authorizations`
**Header：** `Authorization: Bearer <agent_token>`

```json
{
  "game_project_id": 1,
  "valid_until":     null
}
```

**返回：**
```json
{
  "id":                 1,
  "user_id":            1,
  "game_project_id":    1,
  "game_project_code":  "yeya",
  "game_project_name":  "椰芽",
  "status":             "active",
  "valid_until":        null
}
```

## 第八步：验证登录（PC 中控 / 安卓脚本）

### 方式 A：Swagger 直接测试

**接口：** `POST /api/auth/login`

```json
{
  "username":           "user_001",
  "password":           "User@2026!",
  "project_uuid":       "07238db5-129a-4408-b82a-e025be4652a1",
  "device_fingerprint": "test_fp_001",
  "client_type":        "android"
}
```

**预期返回（登录成功）：**
```json
{
  "access_token":      "eyJ...",
  "refresh_token":     "eyJ...",
  "expires_in":        900,
  "user_id":           1,
  "username":          "user_001",
  "user_level":        "normal",
  "game_project_code": "yeya"
}
```

**登录失败时的常见错误：**

| HTTP | detail | 原因 |
|------|--------|------|
| 401 | 用户名或密码错误 | 密码不对 |
| 404 | 游戏项目不存在或已下线 | project_uuid 不匹配 |
| 403 | 无此游戏的授权 | 未做第七步 |
| 403 | 账号已过期 | expired_at 已到期 |
| 403 | 设备绑定数量已达上限 | max_devices 已满 |

### 方式 B：启动 PC 中控

```powershell
cd "E:\Hive-GreatSage\蜂巢·大圣（Hive-GreatSage）项目\HiveGreatSage\HiveGreatSage-PCControl"
conda activate TZYMIR
python main.py
```

在登录窗口填入 `user_001` / `User@2026!` → 点登录。

## 第九步：验证后台数据

### 查看登录日志

**接口：** `GET /admin/api/login-logs/`
**Header：** `Authorization: Bearer <admin_token>`

应看到刚才登录的记录，`success: true`。

### 查看设备绑定

**接口：** `GET /admin/api/users/{user_id}/devices`
**Header：** `Authorization: Bearer <admin_token>`

应看到 `device_fingerprint: "test_fp_001"` 的绑定记录，`status: "active"`。

### 仪表盘概览

**接口：** `GET /admin/api/dashboard`
**Header：** `Authorization: Bearer <admin_token>`

```json
{
  "admin":           "admin",
  "total_users":     1,
  "total_agents":    1,
  "active_projects": 1
}
```

## 管理操作速查

### 查看所有用户
```
GET /api/users/
Authorization: Bearer <admin_token 或 agent_token>
```

### 查看用户详情（含授权列表）
```
GET /api/users/{user_id}
```

### 暂停/恢复用户
```
PATCH /api/users/{user_id}
Body: {"status": "suspended"}   或   {"status": "active"}
```

### 撤销用户授权
```
DELETE /api/users/{user_id}/authorizations/{auth_id}
```

### 查看代理树形结构
```
GET /api/agents/tree
Authorization: Bearer <admin_token>
```

### 查看平台统计
```
GET /api/stats/platform
Authorization: Bearer <admin_token>
```

### 解绑设备
```
DELETE /admin/api/users/{user_id}/devices/{binding_id}
Authorization: Bearer <admin_token>
```
