# Verify 审计日志系统设计

## 一、设计目标

审计日志是平台级不可变操作记录，用于：
- 追溯管理员/代理/用户的关键操作
- 排查安全事件
- 财务对账辅助
- 合规审计

## 二、数据模型 (`app/models/main/audit.py`)

```text
AuditLog:
  id              BIGINT PK
  actor_type      VARCHAR(16)  — admin / agent / user / system
  actor_id        INT          — actor_type 对应的 ID，可为 NULL
  action          VARCHAR(128) — 操作标识，如 "auth.admin.login_success"
  target_type     VARCHAR(64)  — 操作目标类型
  target_id       VARCHAR(64)  — 目标 ID（字符串，兼容 UUID）
  summary         TEXT         — 可读摘要
  metadata_json   JSONB        — 结构化元数据（自动脱敏）
  request_id      VARCHAR(64)  — X-Request-ID
  ip_address      INET         — 客户端 IP
  user_agent      TEXT         — User-Agent
  created_at      TIMESTAMPTZ
```

索引：`actor_id+action`, `target_type+target_id`, `request_id`, `created_at`。

## 三、服务层 (`app/services/audit_service.py`)

### 写入接口

```python
await create_audit_log(
    db=db,
    actor_type="admin",      # admin / agent / user / system
    actor_id=current_admin.id,
    action="user.create",    # 操作标识
    target_type="user",      # 目标类型
    target_id=user_id,       # 目标 ID
    summary="创建用户 xxx",   # 可读摘要
    metadata={...},          # 元数据（自动脱敏）
    request_id=None,         # 自动从 RequestIdMiddleware 获取
    ip_address="1.2.3.4",
    user_agent="...",
)
```

### 敏感字段自动脱敏

`_sanitize_metadata()` 递归遍历 metadata JSON：
- **精确匹配**: `authorization`, `file_data`, `file_content`
- **关键词匹配**: `password`, `passwd`, `pwd`, `token`, `secret`, `api_key`, `apikey`, `private_key`, `credential`, `credentials`
- **后缀匹配**: `_key`, `_secret`, `_token`, `_password`, `_hash`

匹配到的值统一替换为 `<redacted>`。

注意：`_hash` 后缀列入敏感词，因此 `device_fingerprint_hash`、`imsi_hash` 等审计元数据中的哈希值也会被脱敏。这是有意为之——审计日志只保存可关联标识，不保存可反查原文。

## 四、RequestId 链路 (`app/core/middleware.py`)

每个 HTTP 请求由 `RequestIdMiddleware` 生成或透传 `X-Request-ID`：
- 客户端可透传（仅接受 `[A-Za-z0-9._:-]{1,64}` 格式）
- 不符合格式或缺失时自动生成 UUID hex
- 存储到 `ContextVar`，`audit_service.create_audit_log()` 自动读取
- 写回响应头

## 六、写入规范

1. **事务边界**: `create_audit_log` 只 flush，不 commit。由调用方事务统一控制。
2. **失败不阻断**: 审计日志写入失败不应阻断主业务。当前实现随主事务一起成功/回滚。
3. **不记录敏感原文**: 密码、Token、密钥从不进入审计日志。
4. **request_id 优先级**: 显式传入 > `ContextVar` > `None`。
