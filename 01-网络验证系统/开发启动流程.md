# HiveGreatSage-Verify 开发启动流程

> **每次开发前按此顺序执行，缺一不可。**
> 需要同时用到：PowerShell（Windows 侧）和 WSL 终端（Linux 侧）。

---

## 启动顺序总览

```
① WSL 终端  — 启动 PostgreSQL
② WSL 终端  — 启动 Redis
③ PowerShell — 激活 conda 虚拟环境
④ PowerShell — 启动 FastAPI 后端（uvicorn）
⑤ PowerShell — 启动前端开发服务器（vite）
⑥ PowerShell — 启动 Celery Worker（按需）
⑦ PowerShell — 启动 Celery Beat（按需）
```

## § 1 — 启动 PostgreSQL（WSL 内）

> 打开 WSL 终端（Windows Terminal → Ubuntu / Debian）

```bash
# 启动 PostgreSQL 服务
sudo service postgresql start

# 验证是否启动成功
sudo service postgresql status

# 连接验证（必须加 -h 127.0.0.1，走 TCP 而非 Unix socket）
# 不加 -h 则走 peer 认证（要求 OS 用户名 = 数据库用户名）导致失败
psql -h 127.0.0.1 -U hive_user -p 15432 -d hive_platform -c "SELECT 1;"
# 输入密码: hive_dev_2026
# 返回 "?column? = 1" 说明正常
```

> ⚠️ **Peer 认证说明**
> `psql` 不加 `-h` 时走 Unix socket，PostgreSQL 对 socket 连接使用 `peer` 认证方式（要求 OS 用户名 == 数据库用户名）。
> 因为 OS 用户是 `hiveadmin` 而不是 `hive_user`，所以不加 `-h` 会报 `Peer authentication failed`。
> FastAPI 后端连接字符串已含 `127.0.0.1`，走 TCP，不受此影响，后端多正常。

> ⚠️ **如果 TCP 连接也失败**，需要修改 `pg_hba.conf`：
> ```bash
> sudo nano /etc/postgresql/16/main/pg_hba.conf
> # 在文件末尾添加：
> # host  all  hive_user  127.0.0.1/32  scram-sha-256
> sudo service postgresql restart
> ```

**停止命令（结束开发时）：**
```bash
sudo service postgresql stop
```

## § 2 — 启动 Redis（WSL 内）

> 同一个 WSL 终端，或新开一个 WSL 标签页

```bash
# 启动 Redis（指定端口 16379）
redis-server --port 16379 --daemonize yes

# 验证
redis-cli -p 16379 ping
# 返回 PONG 说明正常

# 查看 Redis 进程
ps aux | grep redis
```

> ⚠️ **如果启动时出现 `WARNING Memory overcommit must be enabled`**，运行以下命令修复（不修复也能运行，但建议修复）：
> ```bash
> # 即时生效
> sudo sysctl vm.overcommit_memory=1
> # 永久生效（重启 WSL 后也保留）
> echo 'vm.overcommit_memory = 1' | sudo tee -a /etc/sysctl.conf
> ```

**停止命令：**
```bash
redis-cli -p 16379 shutdown
```

## § 12 — 常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| `Connection refused 15432` | WSL PostgreSQL 未启动 | `sudo service postgresql start` |
| `Connection refused 16379` | WSL Redis 未启动 | `redis-server --port 16379 --daemonize yes` |
| `InvalidPasswordError` | hive_user 权限未配置 | 运行 `scripts/setup_game_db.sql` |
| 前端 `5173` 打不开 | Vite 未启动 | `cd frontend && npm run dev` |
| 前端 API 请求 `Network Error` | 后端 8000 未启动 | `uvicorn app.main:app --port 8000 --reload` |
| `uvicorn: command not found` | 虚拟环境未激活 | `conda activate TZYMIR` |
| Celery `Cannot connect to redis` | Redis 未启动 | 先在 WSL 启动 Redis |
