# 密钥与授权码索引

## 当前定位

本文是项目级敏感凭据索引，不是密钥存储文件。

本文只记录密钥类别、用途、保管位置、风险等级、当前状态和处理建议。本文不记录真实密钥、授权码、Token、数据库密码。真实值只应存在于本机 `.env`、`local.yaml`、密码管理器或 CI/CD Secret 中。

原 `密钥与授权码.md` 已被替换为本索引，原因：不适合作为普通 Markdown 明文文档长期保存。

---

## 总原则

1. 任何真实密钥不得进入 Obsidian 普通 Markdown。
2. 任何真实密钥不得进入 Git 仓库。
3. `.env` 只允许存在于本机或服务器环境，不允许提交。
4. `.env.example` 只允许包含占位符和示例值，不允许包含生产值。
5. 开发期本地凭据如果仅限本机、本地数据库、本地 Redis，可标记为"待确认是否需要更换"，不默认要求立即轮换。
6. 生产凭据、外部服务凭据、公网访问凭据、CI/CD Secret、GitHub Token、服务器 SSH 私钥如进入仓库或对话，应视为高风险，建议立即废弃或轮换。
7. AI 对话中不得复述真实密钥。
8. 日志、测试报告、联调报告不得打印完整 Token、授权码或密码。

## 三、泄露处理流程

处理敏感凭据泄露时，按以下分级执行：

### 3.1 先判断凭据类型

- 开发期本地凭据
- 生产凭据
- 外部服务凭据（如 GitHub Token、Sentry DSN、OCR 授权）
- 公网访问凭据（如 VPS root 密码、云控制台密码）
- CI/CD / GitHub / SSH 凭据

### 3.2 开发期本地凭据

1. 不在对话或文档中复述真实值。
2. 从 Git 当前 HEAD 移除（确认 .gitignore 生效）。
3. 确认仅限本机 localhost 访问。
4. 进入联调/部署前统一更换。
5. 如仅本地使用，可不立即轮换。

### 3.3 生产 / 外部服务 / 公网凭据

1. 停止传播（停止在对话、文档、截图、日志中引用）。
2. 立即废弃或轮换（创建新凭据、废弃旧凭据）。
3. 使旧 Token / 旧连接失效（如 revoke GitHub Token、删除旧 SSH key）。
4. 检查扩散范围：Git 历史、Google Drive、截图、日志、AI 对话记录。
5. 更新本索引和治理清单。
6. 如影响生产服务，按故障处理流程通知相关人员。

### 3.4 CI/CD / GitHub / SSH 凭据（高风险子类）

属于生产/外部凭据，但因其可导致仓库或服务器被控，需额外重视：

1. 立即废弃。
2. 检查是否有未授权访问记录（GitHub 安全日志、服务器 auth.log）。
3. 重建凭据后更新所有依赖该凭据的配置。
4. 确认 Git 历史中是否残留（如曾提交后删除，需 git filter-branch 或 BFG 清理）。

## 四、禁止事项

- 禁止在 Obsidian 普通文档中粘贴真实密钥。
- 禁止把 `.env` 作为知识库附件长期保存。
- 禁止在对话提炼中复述真实授权码、Token、数据库密码。
- 禁止截图展示密钥。
- 禁止把 `.env.example` 示例值用于生产。
- 禁止把日志、测试报告、联调报告中包含的 Token 原样沉淀进 Obsidian。
- 禁止把"开发期本地密码"直接写成"已泄露必须轮换"，除非确认其连接生产/公网/外部服务。
- 禁止把"待确认是否需要更换"写成"已更换"。

## 五、待确认清单

以下事项需要在后续治理中逐一确认：

- Verify 真实 `.env` 是否已从当前 HEAD 移除。
- Verify Git 历史中是否仍有 `.env` 提交。
- 当前密码是否仅用于本地开发（localhost / 127.0.0.1）。
- SECRET_KEY 是否需要在联调/部署前统一重置。
- 数据库密码是否仅限本机访问。
- Redis 是否生产启用密码。
- S3 / MinIO 是否启用真实外部服务（还是本地 MinIO 测试）。
- AndroidScript 仓库是否需要补 `.gitignore`。
- AndroidScript 脚本中是否硬编码了账号密码或 Token。
- PCControl 本地配置文件是否曾进入 Git 仓库。
- PCControl `local.yaml` 是否已在 `.gitignore` 中。
- Google Drive / 本地附件中是否仍有旧密钥文档（如旧 `密钥与授权码.md`、含密码的 `.docx`）。
- CI/CD Secret 是否已建立。
- GitHub Token 是否曾在对话或附件中暴露。
- SSH 私钥是否仅限于开发者本机。
