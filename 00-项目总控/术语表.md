# 术语表

## 当前定位

本文是蜂巢·大圣项目级通用术语表。

专题详细术语仍由对应专题文档维护。本文用于统一跨窗口、跨 AI、跨子系统沟通时的词义。

## 使用规则

1. 已确认术语标记为"✅ 已确认"。
2. 推断术语标记为"⚠️ 需补证据"。
3. 待补证据术语标记为"🔲 待确认"。
4. 不在本文记录真实密钥、授权码、Token、数据库密码。
5. 术语如与源码、测试、运行日志冲突，以源码、测试、运行日志为准。

## 十二、状态词规范

以下为项目文档治理标准状态词。全文统一使用，不得出现同义不同词。

| 状态词 | 含义 | 适用场景 |
|---|---|---|
| 已确认 | 已有文档、代码、测试或用户明确输入支撑的事实 | 决策结论、实现状态、术语定义 |
| 待确认 | 尚未核实、需要补充证据的表述 | 插件安装状态、仓库部署状态、未验证断言 |
| 推断 | 基于已有信息的合理推测 | 从代码行为推断的设计意图 |
| 建议方案 | AI 或开发者提出的可选方案 | 待决策清单、方案讨论 |
| 已实现 | 代码存在且通过测试验证 | 路由、模型、功能模块 |
| 源码存在 | 代码文件存在但未验证运行结果 | 源码精读发现的功能、API、字段 |
| 测试通过 | 对应测试已运行并通过 | pytest 全绿、联调成功 |
| 联调通过 | 多端（后端+前端+中控+安卓）协同验证通过 | 跨子系统功能 |
| 生产可用 | 已完成部署、压测和监控闭环 | Phase 3+ 交付状态 |
| 已删除 | 文件/代码/模型已被移除 | 删除操作记录 |
| 已迁移 | 内容已从旧位置移动到新位置 | 文档拆分、重构迁移 |
| 已废弃 | 历史有价值但当前已不再使用 | 旧架构、旧方案记录 |

重要：**源码存在 ≠ 生产可用**。源码存在只表示文件在仓库中，生产可用需要经过测试、联调、部署、压测、监控全部闭环。

## 十二点一、证据等级术语

| 等级 | 名称 | 含义 | 不得误读为 |
|---|---|---|---|
| D0 | 仅有草稿描述 | 只有方向、占位或草稿入口 | 不代表已设计完成 |
| D1 | 文档证据 | 已有 Obsidian 或专题文档描述 | 不代表源码存在 |
| C1 | 源码证据 | 已精读源码并确认模块、接口、字段或调用链存在 | 不代表测试通过 |
| T1 | 测试文件证据 | 已看到测试文件或测试清单 | 不代表测试实际运行通过 |
| R1 | 单仓运行证据 | 已实际运行单仓测试并通过 | 不代表跨端联调通过 |
| E2 | 跨端联调证据 | 已完成后端、PC、安卓等跨端联调 | 不代表真实设备长期稳定 |
| M1 | 实机验证证据 | 已完成真实 PC / 安卓实机验证 | 不代表生产环境长期可用 |

没有 R1 / E2 / M1 证据时，不得写"已闭环""可发布""生产可用"。

历史矩阵记录、文档标记完成、源码文件存在，都不能自动升级为当前闭环证据。
