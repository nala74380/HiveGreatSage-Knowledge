# 治理拆分承接区

## 当前定位

本目录用于承接 `00-项目总控` 下需要拆分的文档。

核心规则：

```text
拆分前必须先建立对应文件夹，再逐步拆分。
禁止直接在 00-项目总控 根目录下散落多个拆分文件。
```

## 使用规则

每个被拆分的源文档，先在本目录下建立一个同名或语义明确的子文件夹。

示例：

```text
00-项目总控/治理拆分承接区/
├── 工作流方案拆分/
│   ├── README.md
│   ├── 工具分工.md
│   ├── AI协作规范.md
│   ├── 对话提炼流程.md
│   └── 历史搭建指南.md
└── 编码规范拆分/
    ├── README.md
    ├── 代码规范.md
    ├── 文档规范.md
    └── Git提交规范.md
```

## 拆分流程

```text
1. 先建立子文件夹与 README.md。
2. README.md 写明来源文档、拆分目标、非目标、迁移计划。
3. 每次只迁移一个职责块。
4. 源文档在迁移完成前继续保留为主入口。
5. 迁移完成后，源文档改为导航页或过时跳转页。
6. 所有合并、拆分、删除必须回写 文档治理清理清单.md。
```
