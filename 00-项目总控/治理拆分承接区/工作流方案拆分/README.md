# 工作流方案拆分

## 当前任务理解

本目录用于承接 `00-项目总控/工作流方案.md` 的逐步拆分。

源文档原本同时包含：

```text
1. 工具角色定义
2. 数据流向总图
3. 日常工作循环
4. 工具补充建议
5. 对话提炼模板
6. Obsidian 插件清单
7. 从零搭建步骤
8. 常见问题
9. 工作流价值总结
```

这些内容都与“工作流”有关，但职责不同。继续全部放在一个文档中，会导致主文档越来越像历史教程，不利于长期维护。

## 迁移原则

```text
1. 每次只迁移一个职责块。
2. 迁移后新文档必须有 YAML frontmatter。
3. 新文档必须保留来源说明。
4. 原文档在迁移完成前继续作为主入口。
5. 全部迁移完成后，原 工作流方案.md 改为导航页。
6. 每轮迁移必须回写 文档治理清理清单.md。
```
