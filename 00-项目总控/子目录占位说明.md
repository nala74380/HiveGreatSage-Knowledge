## _模板/

存放各种文档的通用模板，新建文档时复制即可。

规划文档：

```
_模板/
├── 对话提炼模板.md          ← 每次 Claude 对话结束后提炼结论用
├── 架构设计模板.md          ← 新子系统架构文档的起点
├── 问题追踪模板.md          ← 遇到棘手问题时使用
└── 游戏适配模板.md          ← 新游戏接入时使用
```

关于"对话提炼模板"的内容参见 [[工作流方案]] 第五节。
建议优先创建这三个模板（对话提炼、架构设计、问题追踪），降低未来产出文档的摩擦。

## 99-Claude对话提炼/

每次有价值的 Claude 对话结束后，花 5-15 分钟在这里产出一份提炼文档。

命名规则：`YYYY-MM-DD_主题.md`

示例：

```
99-Claude对话提炼/
├── 2026-04-13_工作流确定.md
├── 2026-04-13_项目总大纲初稿.md
├── 2026-04-14_技术栈锁定.md
├── 2026-04-16_PC安卓通信方案.md
└── 2026-04-16_文档审查.md        ← 本次审查应产生一份
```

这是整个工作流最容易被忽略但最重要的环节（参见 [[工作流方案]] 的 Phase 4 说明）。
