
## 二、Step 1：创建 Obsidian Vault

历史建议：

```text
1. 打开 Obsidian。
2. 创建新 Vault。
3. 命名为 HiveGreatSage。
4. 按模块创建基础目录结构。
5. 在 00-项目总控 下创建项目总大纲、工作流方案、AI安全术语表、决策日志。
6. 在 _模板 下创建对话提炼模板、架构设计模板、问题追踪模板。
```

建议的基础目录：

```text
HiveGreatSage/
├── 00-项目总控/
├── 01-网络验证系统/
├── 02-PC中控框架/
├── 03-安卓脚本框架/
├── 04-官网监控/
├── 05-工具与拓展/
├── 06-游戏适配/
├── 99-Claude对话提炼/
└── _模板/
```

当前治理说明：

```text
1. 该结构是早期规划结构。
2. 当前实际结构必须以 GitHub 中 HiveGreatSage-Knowledge 为准。
3. 如果实际仓库没有某些目录，不应凭本文断言它们存在。
```

## 三、Step 2：安装 Obsidian 插件

历史建议安装：

```text
1. Obsidian Git。
2. Excalidraw。
3. Kanban。
4. Templater。
```

历史配置建议：

```text
1. 关闭安全模式。
2. 安装所需第三方插件。
3. Obsidian Git 连接 HiveGreatSage-Knowledge 私有仓库。
4. 设置自动提交或手动提交策略。
5. Templater 指向 _模板 目录。
```

当前治理说明：

```text
1. 插件是否已安装属于环境事实，本文不确认。
2. 当前插件建议应参考 Obsidian配置与插件.md。
3. 不建议为了还原历史步骤而安装不需要的插件。
```

## 四、Step 3：创建 GitHub 仓库

历史建议创建：

```text
HiveGreatSage-Knowledge
HiveGreatSage-Verify
HiveGreatSage-PCControl
HiveGreatSage-AndroidScript
HiveGreatSage-Monitor
HiveGreatSage-Tools
```

历史分支建议：

```text
main = 稳定版本
开发分支 = 日常开发或功能验证
feature/xxx = 功能分支
fix/xxx = 修复分支
```

当前治理说明：

```text
1. 当前已实际参与治理的仓库包括 Knowledge、Verify、PCControl、AndroidScript。
2. Monitor 和 Tools 是否存在、是否启用、是否为空，需以 GitHub 实际仓库为准。
3. 分支策略是否已执行，需以 GitHub 分支为准。
```

## 五、Step 4：配置编辑器分工

历史建议：

```text
Cursor:
  - HiveGreatSage-AndroidScript
  - HiveGreatSage-Tools
  - 快速 AI 辅助原型

PyCharm:
  - HiveGreatSage-Verify
  - HiveGreatSage-PCControl
  - HiveGreatSage-Monitor
```

核心规则：

```text
不要两个编辑器同时打开同一个项目。
不要两个工具同时改同一个文件。
```

当前治理说明：

```text
该建议仍有长期价值，但属于工作习惯规范，不证明当前本地环境已经如此配置。
```

## 六、Step 5：配置 AI Project

历史建议：

```text
1. 创建 Claude / ChatGPT Project。
2. 上传或同步项目总大纲。
3. 上传或同步 AI 安全术语表。
4. 上传当前模块架构设计文档。
5. 设置窗口命名规则。
```

建议窗口命名：

```text
[模块编号]-[主题]-[日期MMDD]
```

示例：

```text
02-PC中控-通信协议设计-0413
03-安卓脚本-OCR方案选型-0415
00-工作流-整体复盘-0420
```

```text
当前 AI 协作规范已经拆入 AI协作规范.md。
历史搭建步骤仅保留为来源参考。
```

## 七、Step 6：Google Drive 文件夹

历史建议：

```text
1. 在 Google Drive 创建 HiveGreatSage-Docs。
2. 设置子文件夹：对外文档 / 备份 / 参考资料。
3. 将 Google Drive 作为对外镜像，不作为日常编辑位置。
```

当前治理说明：

```text
1. Google Drive 是否实际建立，需用户或连接器确认。
2. 当前数据流向仍应坚持 Obsidian 定稿 → Google Drive 镜像。
3. 不允许 Google Drive 反向覆盖 Obsidian，除非做受控迁移。
```

## 八、Step 7：draw.io 配置

历史建议：

```text
1. 使用 draw.io 桌面版或插件。
2. 图文件保存到对应 Obsidian 模块文件夹。
3. 同时导出 PNG / SVG 供文档嵌入。
```

当前治理说明：

```text
1. draw.io 用于正式架构图、ER 图、时序图。
2. Excalidraw 用于推演草图。
3. 图不能单独成为事实源。
4. 图过时后必须在对应文档中标记。
```
