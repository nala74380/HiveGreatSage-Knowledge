## 二、版本号规范

### 格式

```text
V主版本.次版本.修订号

V1.0.0  ← 初始发布
V1.0.1  ← 修复 bug / 小改动
V1.1.0  ← 新增功能（向后兼容）
V2.0.0  ← 重大变更（可能不兼容旧版）
```

### 版本号适用范围

|层级|说明|
|---|---|
|**项目版本**|整个子系统的发布版本（如 PC中控 V1.2.0）|
|**文件版本**|单个文件的版本（文件头中标注）|

文件版本独立于项目版本。一个文件可能在项目 V1.0.0 到 V1.2.0 之间经历了多次修改，文件版本会高于项目初始版本。

## 四、注释规范

### 原则

```text
1. 注释说明"为什么"，不说明"是什么"
   ✗ # 将 count 加 1
   ✓ # 重试次数用完后不再尝试，避免无限循环

2. 复杂逻辑必须注释
   · 正则表达式
   · 位运算
   · 非显而易见的业务规则
   · 临时绕过的 bug（标注 TODO/FIXME）

3. 不注释显而易见的代码
   ✗ self.username = username  # 设置用户名
```

### TODO / FIXME / HACK 标记

```python
# TODO: 后续需要实现 Token 自动刷新 —— 2026-04-16
# FIXME: 当设备数超过 200 时列表渲染卡顿 —— 2026-04-20
# HACK: 临时绕过懒人精灵 OCR 的编码问题 —— 2026-05-01
```

统一格式：`# 标记: 说明 —— 日期`

### Python 函数文档字符串

```python
def login(self, username: str, password: str) -> LoginResult:
    """用户登录验证。

    调用云端 API 进行身份验证，成功后保存 Token 和用户信息。

    Args:
        username: 用户名

    Returns:
        LoginResult: 包含 success、token、user_info、error_message

    Raises:
        NetworkError: 网络连接失败时抛出
    """
```

## 八、错误处理规范

### Python

```python
# ✗ 错误：捕获所有异常然后沉默
try:
    result = api.get_devices()
except:
    pass

# ✓ 正确：捕获具体异常，记录日志，合理处理
try:
    result = api.get_devices()
except httpx.TimeoutException:
    logger.warning("获取设备列表超时，将使用缓存数据")
    result = cache.get("device_list")
except httpx.HTTPStatusError as e:
    if e.response.status_code == 401:
        self.token_expired.emit()
    else:
        logger.error(f"获取设备列表失败: {e}")
        raise
```

### Lua

```lua
-- ✗ 错误：不检查返回值
local data = http_get(url)
process(data)

-- ✓ 正确：检查返回值，处理失败
local ok, data = pcall(http_get, url)
if not ok then
    log_error("请求失败: " .. tostring(data))
    return nil, "network_error"
end
if not data then
    log_error("返回数据为空")
    return nil, "empty_response"
end
process(data)
```

## 九、日志规范

### 日志级别使用原则

```text
DEBUG:    开发调试信息（生产环境关闭）
          示例: 请求参数、响应原文、变量值
INFO:     正常运行的关键节点
          示例: 登录成功、同步完成、任务开始/结束
WARNING:  非致命异常，程序可继续运行
          示例: 请求超时已重试、使用缓存数据
ERROR:    功能受损，需要关注
          示例: API 持续不可达、数据解析失败
CRITICAL: 程序无法继续运行
          示例: Token 无法刷新、配置文件损坏
```

### 日志格式

```text
Python:
  2026-04-16 14:30:25 [INFO] [auth] 登录成功: user=Hailin level=vip
```

## 十、安全规范

```text
1. 密码/Token 不明文存储
   · 密码传输前加密
   · Token 本地存储使用 cryptography 加密
   · 日志中不打印密码和完整 Token

2. API 通信使用 HTTPS
   · 不使用 HTTP（即使是开发环境也建议 HTTPS）
   · 验证服务端 SSL 证书

3. 配置文件中的敏感信息
   · 数据库密码、API 密钥等放在 local.yaml 或 .env
   · 这些文件必须在 .gitignore 中

4. SQL 注入防护
   · 使用 ORM（SQLAlchemy）或参数化查询
   · 永远不要拼接 SQL 字符串
```
