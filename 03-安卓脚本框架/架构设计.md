# 子系统3：安卓端脚本框架 — 架构设计

> **代号：** AndroidScript **仓库：** HiveGreatSage-AndroidScript（框架基座） **编辑器：** Cursor（代码编写）+ 懒人精灵 Win IDE（调试 / 打包） **运行环境：** 懒人精灵高级版 **root / 激活模式**（无障碍模式不支持，全部删除） **脚本语言：** Lua 5.3 **产出：** 每个游戏 fork 基座后，打包为独立 `.lrj`（热更新包）或 `.apk`

---

## 一、已确认的核心决策

|决策项|结论|
|---|---|
|运行模式|仅 root / 激活，无障碍分支全部删除|
|OCR 引擎|TomatoOCR 收费版，APK 方式加载（`LuaEngine.loadApk`）|
|YOLO 引擎|TomatoYOLO 收费版，同 OCR 方式加载（可选模块）|
|开发基准分辨率|**1280 × 720**，兼容 1920×1080 / 2560×1440，统一 DPI 320|
|分辨率适配|三档均 16:9，`setScreenScale` 线性缩放无畸变，截图模板一套通用|
|UI 方案|动态 UI（`ui.*` API）+ H5 WebView，游戏定制层按需选择|
|启动 UI|固定两页：**登录页**（账号/密码/编号/倒计时/随机延时）+ **设置页**（中控IP/全局初始化）|
|局域网通信|WebSocket（`startWebSocket`，仅 `ws://`，不支持 `wss://`）|
|热更新格式|`.lrj`（`installLrPkg`，Lua 区分大小写，拼错直接报错）|
|代理/VPN|框架内置多客户端支持，`proxy.lua` 配置驱动，用户按需切换|
|仓库策略|独立仓库模式，每个游戏 fork 框架基座|

## 二、仓库层级与 IDE 目录结构

懒人精灵高级版 IDE 强制四个顶级目录，**GitHub 仓库根目录即懒人精灵项目根目录**，两者是同一层级，不额外嵌套。

`require()` 路径基准为 `getWorkPath()`（即 `脚本/`），支持子目录：

```lua
require("framework/verify")      -- 加载 脚本/framework/verify.lua
require("game/tasks/daily")      -- 加载 脚本/game/tasks/daily.lua
```

### 4.2 倒计时逻辑说明

```text
初次登录：1200s（20分钟）
  适用于：脚本首次启动，等待大批设备错峰
  归零后：自动调用登录流程

二次登录：300s（5分钟）
  适用于：中途退出后重进，无需等太久
  判断依据：本地存储 last_login_time，启动时计算间隔

随机延时（勾选时）：
  登录成功后，sleep(rnd(1000, 60000))
  用途：大型工作室几百台设备，避免同一秒发起相同请求
  配置写入：Config.RANDOM_DELAY_ENABLED / MIN / MAX
```

### 4.3 全局初始化说明

```text
触发场景：
  账号被封禁 → 换新账号重新上号
  历史游戏配置（如服务器选择/快捷键/角色位置等）存于游戏本地
  → 可能被检测到异常 → 需要清除后以"全新状态"登录

执行内容（游戏定制层实现，框架层预留接口）：
  game/ui_helper.lua → on_global_init()
    1. 清除游戏应用数据（exec("pm clear " .. GAME_PACKAGE)）
    2. 清除本地脚本缓存（readKeyVal/writeKeyVal 记录清零）
    3. 重置 config 中的运行时状态字段

UI 流程：
  点击"执行全局初始化" → 弹出二次确认："此操作将清除所有游戏本地数据，确认执行？"
  → 确认 → 执行 → toast提示 → 重启脚本
  → 取消 → 关闭弹窗，不执行
```

### 5.2 `framework/proxy.lua` — 代理/VPN 多客户端管理

**设计原则：** 注册表 + 钩子函数，框架层只负责"启动App → 等待 → Home"，具体的"点击连接按钮"由游戏定制层的 `game/ui_map.lua` 注册。

```text
已支持客户端注册表：

  Key          名称              包名
  ─────────────────────────────────────────────────────────────
  99           99手游加速器      com.excean.na
  biubiu       BiuBiu加速器      com.njh.biubiu
  kitsunebi    Kitsunebi代理     fun.kitsunebi.kitsunebi4android
  openvpn      OpenVPN           de.blinkt.openvpn
  ─────────────────────────────────────────────────────────────
  （预留）uu   UU加速器          com.netease.uu
  （预留）xunyou 迅游加速器      com.xunyou.mobile
  （预留）nn   NN加速器          com.nn.gameacc
  （预留）v2ray v2rayNG          com.v2ray.ang
  （预留）clash ClashMeta        com.github.metacubex.clash.meta

公共接口：
  Proxy.start()                  按 Config.PROXY_CLIENT 启动代理
  Proxy.stop()                   停止代理
  Proxy.is_installed()           检查是否已安装
  Proxy.register_start_fn(key, fn)  注册"点击连接"操作（游戏定制层调用）
  Proxy.list()                   列出所有已知客户端（调试用）
```

新增客户端只需在 `PROXY_CLIENTS` 表追加一条记录，其余代码无需修改。

### 5.3 `framework/ocr.lua` — TomatoOCR 适配器

**加载方式（已确认）：**

```lua
-- APK 放入 资源/.rc 文件内，loadApk 直接从 .rc 内加载，无需 extractAssets
local loader = LuaEngine.loadApk("TomatoOCR.apk")
local OCRApi  = loader.loadClass("com.tomato.ocr.lr.OCRApi")
local engine  = OCRApi.init(LuaEngine.getContext())
engine.setLicense(Config.OCR_LICENSE)
engine.setRecType(Config.OCR_REC_TYPE)
engine.setDetBoxType("rect")
engine.setDetUnclipRatio(1.9)
engine.setRecScoreThreshold(0.3)
engine.setReturnType("json")
engine.setBinaryThresh(0)
```

**对上层暴露的接口（上层业务只调这三个）：**

```lua
OCR.init()                     → boolean   初始化（main.lua调一次）
OCR.detect(l, t, r, b)        → table[]   区域识别，返回 {label,x,y,score}
OCR.find_text(text, l,t,r,b)  → x, y      查找文字，失败返回 -1,-1
OCR.get_text(l, t, r, b)      → string    获取区域全部文字拼接
```

⚠️ 所有 Bitmap 操作必须手动 `bmp.recycle()`，否则内存泄漏。

### 5.5 `main.lua` — 标准启动序列

```lua
-- =========== 依赖加载 ===========
local Config       = require("config")
local Screen       = require("framework/screen")
local Proxy        = require("framework/proxy")
local OCR          = require("framework/ocr")
local Verify       = require("framework/verify")
local Updater      = require("framework/updater")
local CloudSync    = require("framework/cloud_sync")
local Scheduler    = require("framework/scheduler")
local Logger       = require("framework/logger")
local ErrorHandler = require("framework/error_handler")

-- =========== 停止回调 ===========
setStopCallBack(function(err, exitcode)
    Logger.info("停止 err=" .. tostring(err) .. " code=" .. tostring(exitcode))
    releaseCapture()
end)

-- =========== 启动自检（环境探针） ===========
print("=== 启动自检 ===")
print("workPath="   .. getWorkPath())
print("sdPath="     .. getSdPath())
print("apkVer="     .. getApkVerInt())
if getCpuArch then
    local t = getCpuArch()
    print("cpuArch=" .. (t==2 and "arm64" or t==1 and "arm" or "x86"))
end
-- 确认 root/激活 模式，否则退出
if getRunEnvType() ~= 0 then
    toast("请使用 root 或激活模式运行", 0, 0, 20)
    sleep(3000); exitScript()
end
print("runEnv=root/激活 ✓")

-- =========== 显示启动面板（登录/设置） ===========
-- main.ui 阻塞展示，用户完成登录或点击取消后继续
local ui_result = showUI("main.ui", -1, -1, function(handle, event, a1, a2, a3)
    -- UI事件处理（见 4.1 页面结构）
end)

-- =========== 初始化序列 ===========
Screen.init()
Logger.init()

-- 随机延时（大型工作室防同步）
if Config.RANDOM_DELAY_ENABLED then
    local delay = rnd(Config.RANDOM_DELAY_MIN, Config.RANDOM_DELAY_MAX) * 1000
    print("[main] 随机延时 " .. (delay/1000) .. "s")
    sleep(delay)
end

-- 代理/VPN
Proxy.start()

-- OCR 初始化
if not OCR.init() then
    toast("OCR 初始化失败，检查 License", 0, 0, 20)
    sleep(3000); exitScript()
end

-- 授权验证（通过代理网络）
if not Verify.login() then
    toast("授权验证失败", 0, 0, 20)
    sleep(3000); exitScript()
end

-- 热更新检查（有更新时自动 restartScript()）
Updater.check_and_update()

-- 拉取云端参数
CloudSync.pull_params()

-- =========== 启动任务调度器 ===========
Scheduler.run()
```

## 十、⚠️ 已知风险与注意事项

```text
性能热点（长稳测试必查）：
  · keepCapture() 不配套 releaseCapture() → 内存泄漏
  · LuaEngine.snapShot() 不调 bmp.recycle() → 内存泄漏
  · OCR + 截图 + 找色叠加时，瓶颈在整条链路而非单个 API
  · 最大并发线程数 ≤ 10 个（beginThread 限制）

热更新函数名（高频错误）：
  ✓ installLrPkg      ← 正确
  ✗ installLRPkg      ← 错误（Lua 区分大小写，调用失败无报错提示）

倒计时 UI：
  · showUI 期间主线程阻塞，倒计时需要在 UI 事件回调中用 setTimer 驱动
  · setTimer 回调运行在主线程，可以安全调用 exitScript / restartScript

全局初始化：
  · pm clear 会清除应用所有数据包括登录态，执行后需重新走登录流程
  · 必须做二次确认弹窗，防止误触

WebSocket 限制：
  · 只支持 ws://，不支持 wss://（加密连接）
  · 回调函数不能阻塞，不能直接调 exitScript/restartScript
  · 需要在主线程操作时，用 setTimer 转发到主线程执行
```
