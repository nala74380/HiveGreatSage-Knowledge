## 一、Bug 修复记录

### 1.1 路由 404（所有二级路由均 404）

**根本原因**：`sharedRoutes` 和 `adminRoutes` 两棵路由树都用 `path:'/'` 注册父路由，Vue Router 只匹配第一棵树，导致 `users/:id`、`login-logs` 等路径 404。

**修复方案**：重写 `router/index.js`，所有登录后页面**合并进单一** `path:'/'` 父路由，彻底消除双树冲突。不再使用 `routes/shared.js` 和 `routes/admin.js` 分文件（已保留但废弃）。

### 1.2 热更新 500 错误

**根本原因**：`update_admin.py` 调 `get_game_db(code_name)` 连接游戏库；普通验证项目 `db_name=NULL`，连接必然失败。

**修复方案**：版本记录从游戏库迁移至主库，新建 `version_record` 表（Migration 0006），`update_admin.py` 全部改用主库 + `project_id`（Integer），同时支持游戏项目和验证项目热更新。

### 1.3 热更新 422（page_size 超出限制）

前端 `projectApi.list({ page_size: 200 })` 超过后端 `le=100` 限制。  
修复：`projects.py` 将 `le=100` 改为 `le=500`。

### 1.4 Alembic 多 head 冲突

两个迁移文件均以 `0004_...` 为 `down_revision`，链断裂。  
修复：将 `0005_version_record_main` 改为 `0006_version_record_main`，`down_revision` 指向 `0005_add_imsi_to_device_binding`，恢复线性链。

### 1.5 前端 `formatDate` 重复声明

`utils/format.js` 已导出 `formatDate`，`UserList.vue` 内又有本地同名函数定义，Vite 编译报错。  
修复：删除 `UserList.vue` 内的本地重复定义，统一使用 import。

### 1.6 Admin 路由 ImportError

`admin.py` 引入了 `get_current_agent_or_admin`（不存在），`AgentProjectAuth`（未使用）。  
修复：删除这两个无效 import。

## 三、数据库变更

| Migration | 内容 |
|---|---|
| `0003_add_user_max_devices` | user 表加 `max_devices` 字段 |
| `0004_project_type_and_agent_auth` | game_project 加 `project_type`，新建 `agent_project_auth` 表 |
| `0005_add_imsi_to_device_binding` | device_binding 加 `imsi` 字段 |
| `0006_version_record_main` | **主库**新建 `version_record` 表（替代游戏库版本记录） |

### version_record 表（主库）

```sql
CREATE TABLE version_record (
    id              SERIAL PRIMARY KEY,
    game_project_id INTEGER REFERENCES game_project(id) ON DELETE CASCADE,
    client_type     VARCHAR(10) NOT NULL,       -- 'pc' | 'android'
    version         VARCHAR(20) NOT NULL,
    package_path    VARCHAR(500) NOT NULL,
    checksum_sha256 VARCHAR(64),
    release_notes   TEXT,
    force_update    BOOLEAN NOT NULL DEFAULT FALSE,
    is_active       BOOLEAN NOT NULL DEFAULT TRUE,
    released_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE(game_project_id, client_type, version)
);
```

**设计决策**：版本记录统一存主库，不依赖游戏库。游戏项目和普通验证项目均支持热更新。

## 五、前端核心功能改进

### 5.1 用户管理列表增强

- **勾选 + 批量删除**：`el-table` 加 `selection` 列，选中后显示蓝色工具栏，批量调 `DELETE /admin/api/users/{id}`
- **授权项目列**：后端返回 `active_project_names`，列表直接展示项目名 tag，无授权显示"未授权"
- **点击用户名**：跳转 `/devices?user_id=xxx&username=xxx`，设备页自动过滤
- **按项目过滤**：过滤栏加项目选择器，后端通过 Authorization 子查询实现

### 5.2 代理管理列表增强

同用户管理：勾选 + 批量删除。

### 5.3 设备监控支持用户过滤

`DeviceList.vue` 从路由 query 读取 `user_id`，有值时调 `/admin/api/users/{id}/devices`（Admin Token），无值时走原 `/api/device/list`（User Token）。`useDevicePoller` 接受 `userIdRef` 参数，变化时自动重新拉取。

### 5.4 项目代号自动生成

新建项目时按类型前缀（`game_` / `verify_`）+ 时间戳后 6 位自动生成，用户可手动覆盖。

### 5.5 时区统一

- 后端：`app/core/timezone.py` 使用 `zoneinfo`（Python 3.9+ 标准库），默认 `Asia/Shanghai`，`.env` 可配置
- 前端：`utils/format.js` 所有时间格式化函数动态读取 `useSettingsStore().timezone`
- 系统设置页：下拉选择时区，修改后全局即时生效，持久化到 `localStorage`

### 5.6 代理权限闭环

代理授权用户项目时，`user_service.grant_authorization()` 检查 `AgentProjectAuth` 表，代理只能授权自己有效授权的项目（403 拒绝越权）。
