# 未修复问题清单

> 来源：Claude 25轮精读 + ChatGPT 多轮静态审查
> 状态：代码可修项已全部修复，剩余为架构债/运维/环境问题

## 一、运维配置（2 项）

### 1. nginx `/updates/` 缺少 token 校验
- **严重度**：中高
- **文件**：`deploy/nginx.conf`
- **现状**：`location /updates/` 是普通静态 alias，无签名或鉴权
- **说明**：LocalStorage 已实现 JWT 签名下载 URL，但 nginx 侧未配置 `auth_request` 校验
- **建议**：添加 `auth_request /auth` 或使用 `secure_link` 模块

### 2. 旧点数物理表未在迁移中删除
- **严重度**：中
- **迁移文件**：`0014_accounting_center_schema.py` 明确写"暂不删除旧表"
- **旧表**：`agent_balance` / `balance_transaction` / `authorization_charge`
- **现状**：ORM 模型已删除，但数据库物理表可能仍在
- **建议**：单独立决策项（D0xx），确认生产库数据 → 归档 → 建 DROP TABLE 迁移

## 二、架构改进（5 项）

### 3. Router 层偏重，需服务层下沉
- **严重度**：中（架构债）
- **涉及文件**：
  - `app/routers/update_admin.py` — 上传校验/临时文件/SHA-256/落库/缓存/审计过多
  - `app/routers/agents.py` — 1281行，余额/流水/代理树混合
  - `app/routers/agent_scope_management.py` — 1225行，包含转账行锁逻辑
  - `app/routers/accounting.py` — 512行，包含金额校验
- **说明**：`服务层业务规则.md` 已明确 Router 不应包含复杂业务逻辑
- **建议**：Phase 3 单独重构轮次

### 4. 缺少 OpenAPI 导出脚本和快照
- **严重度**：P1
- **现状**：无 `scripts/export_openapi.py`，无 `docs/openapi/*.json`
- **影响**：接口契约无法版本化管理，前后端字段变更不易发现
- **建议**：新增 `scripts/export_openapi.py`，生成 `openapi_YYYY-MM-DD.json`

### 5. 55 个 Router 操作缺少显式 `response_model`
- **严重度**：P1
- **涉及路由**：accounting(15)、agent_scope_management(14)、device_admin(4)、update_admin(3)、pricing(3)、audit_admin(2) 等
- **影响**：OpenAPI Schema 精度不足，返回 `dict`/`list[dict]` 无法描述字段结构
- **建议**：逐路由补齐 Pydantic response_model

### 6. 部分请求 Schema 定义在 Router 内部
- **严重度**：P2
- **涉及**：`accounting.py`(AccountingAmountRequest)、`pricing.py`(SetPriceRequest)、`agent_scope_management.py`(5 classes)
- **建议**：逐步迁移到 `app/schemas/` 对应文件

### 7. 旧前/后端接口对照表需要重生成
- **严重度**：P2
- **文件**：`01-网络验证系统/前后端接口对照表.md`
- **现状**：引用不存在的 `shared/user.js`/`project.js`/`device.js` 等文件
- **建议**：待 OpenAPI 快照生成后，基于真实文件结构重新生成

## 三、环境依赖（1 项）

### 8. 当前环境缺失 Python 依赖，无法运行 pytest / 生成 OpenAPI
- **严重度**：P1
- **现象**：`ModuleNotFoundError: No module named 'sqlalchemy'` 等
- **影响**：无法验证修复效果、无法生成接口快照、无法跑测试基线
- **说明**：此为 ChatGPT 审查环境限制，本地开发环境不一定有同样问题

## 四、前端架构（1 项）

### 9. 前端部分页面旧字段兼容 fallback 可进一步清理（非阻塞）
- **严重度**：P3
- **涉及**：`AgentEditDrawer.vue` 的 `balanceValue()` 函数中仍保留旧字段 fallback 逻辑
- **说明**：该 fallback 在上一轮已简化为从新字段读取，但函数结构可进一步简化
- **建议**：下一次前端重构时一并处理
