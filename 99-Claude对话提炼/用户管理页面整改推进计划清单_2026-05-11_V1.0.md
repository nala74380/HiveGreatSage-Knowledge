# 用户管理页面整改推进计划清单 V1.0

> 本文是基于《用户管理页面推理_风险项_改进计划_2026-05-11_V1.2_完整合并修正版.md》提炼出的**逐步推进整改计划清单**。
>
> 本文不是设计解释文档，而是执行路线文档。
>
> 核心原则：
>
> ```text
> 1. 不在旧结构上零散打补丁。
> 2. 不一次性大爆炸重构。
> 3. 按阶段推进：决策冻结 → 前端小修 → 抽屉增强 → 后端能力 → 账务冻结 → 设备联动 → 测试验收 → 文档同步。
> 4. 每阶段都必须有：范围、非目标、前置条件、执行项、验收标准、回退方式。
> 5. 每做完一阶段，由 ChatGPT 复核，再让 DeepSeek 进入下一阶段。
> ```

---

# 一、当前整改总目标

## 1.1 目标

把当前用户管理页面从：

```text
用户超级列表 + 部分授权编辑 + 部分代理扣点提示
```

逐步治理为：

```text
用户超级列表：
  负责浏览、筛选、摘要、进入编辑。

编辑抽屉：
  负责复杂用户与授权操作。

管理员编辑抽屉：
  完整治理面板，包含用户、授权、设备、账务、冻结、审计、代理归属。

代理编辑抽屉：
  管理员抽屉的受限子集，只允许操作自己客户，且所有付费动作必须扣点预览。

后端：
  使用专用动作接口承载授权新项目、新增设备、续费、升级等级、停用冻结、启用恢复。

账务：
  现有钱包、扣点、返点主线保持清晰；新增授权冻结权益记录，不混用钱包 frozen_credit。
```

---

## 1.2 非目标

本计划当前不做以下事情：

```text
1. 不直接重做整个后台 UI 框架。
2. 不直接引入授权批次 / 设备批次大模型。
3. 不在第一阶段实现新旧设备不同到期时间。
4. 不把所有功能一次性塞进 UserList.vue。
5. 不把授权权益冻结混入 AccountingWallet.frozen_credit。
6. 不绕过账务中心直接修改代理余额。
7. 不让代理通过普通 PATCH 免费修改等级、设备数、到期时间。
```


# 三、阶段 P0：业务决策冻结

## 3.1 阶段目标

在代码改动前，先冻结几条会影响后续所有实现的业务规则。

## 3.2 前置条件

```text
1. 已阅读 V1.2 完整合并修正版。
2. 已确认超级列表操作按钮保持原样。
3. 已确认复杂功能进入编辑抽屉。
4. 已确认管理员抽屉必须比代理端更完整。
```

## 3.3 决策清单

### D027：Agent 是否允许删除用户账户

建议结论：

```text
Agent 不允许删除用户账户。
Agent 只能停用用户状态或停用项目授权。
Admin 删除用户账户才触发返点清算。
```

待确认项：

```text
1. Agent 是否允许“停用用户账号主体”？
2. 停用用户账号是否影响全部项目授权？
3. 停用用户账号是否冻结权益？
```

建议第一版：

```text
Agent 可停用用户账号主体。
停用用户账号不返点。
停用用户账号是否冻结所有项目授权，暂不自动做；先只影响登录状态。
```

---

### D030：启用冻结授权规则

已确认：

```text
方案 A：启用 = 恢复被冻结的剩余权益，不扣点。
```

仍需确认：

```text
1. Agent 是否可自行启用自己停用的授权。
2. Admin 是否可强制启用任意 frozen 授权。
3. 启用前是否必须校验代理项目授权仍有效。
```

建议第一版：

```text
Agent 可启用自己停用且仍属于自己客户的 frozen 授权。
Admin 可强制启用。
启用前必须校验用户、项目、代理、代理项目授权仍有效。
```

---

### D032：授权冻结权益建模方式

建议结论：

```text
新增 AuthorizationFreezeRecord。
不复用 AccountingWallet.frozen_credit。
不复用 LedgerEntry.freeze / unfreeze。
不复用 AuthorizationChargeSnapshot.refund_status。
```

---

### D033：冻结 / 释放是否写 AccountingDocument

建议第一版：

```text
停用冻结：
  写 AuthorizationFreezeRecord + audit_log。
  不写 Wallet。
  不写 Ledger。
  第一版可不写 AccountingDocument。

启用释放：
  更新 AuthorizationFreezeRecord + audit_log。
  不写 Wallet。
  不写 Ledger。
  第一版可不写 AccountingDocument。

删除清算：
  继续走现有 delete_refund document + refund ledger。
```

---

### D034：Admin 停用授权是否冻结权益

建议第一版：

```text
Agent 停用授权：停用 + 冻结权益。
Admin 停用授权：默认仅停用，不自动返点；是否冻结由管理员选项决定。
```

可选实现：

```text
Admin 停用弹窗增加选项：
  1. 仅停用
  2. 停用并冻结权益
```

---

## 3.4 阶段验收标准

```text
1. 决策日志新增 D027、D030、D032、D033、D034。
2. 未修复问题清单同步更新。
3. Knowledge 中明确 Agent 删除用户、停用授权、启用授权、冻结权益的规则。
4. DeepSeek 不再按旧口径实现“Agent 删除用户返点”。
```

## 3.5 回退方式

```text
如果冻结权益模型暂时不实现：
  停用授权仍只能保留当前 status=suspended 逻辑；
  前端不得显示“冻结权益已完成”；
  文案必须标记“冻结权益待接入”。
```

# 十、阶段 P7：授权停用 / 启用 / 冻结权益模型

## 10.1 阶段目标

实现：

```text
Agent 停用授权：
  停用 + 冻结剩余权益 + 不返点。

Agent 启用授权：
  恢复被冻结剩余权益 + 不扣点。

Admin 删除用户：
  清算冻结权益 + 返点。
```

## 10.2 新增模型

```text
AuthorizationFreezeRecord
```

字段：

```text
id
authorization_id
agent_id
user_id
project_id
freeze_type
status
frozen_at
frozen_by_agent_id
frozen_by_admin_id
original_valid_until
remaining_hours
estimated_remaining_points
released_at
released_by_agent_id
released_by_admin_id
new_valid_until
refunded_at
settled_by_admin_id
refund_document_id
remark
created_at
updated_at
```

## 10.3 新增服务

```text
suspend_authorization_with_freeze()
enable_authorization_with_release()
settle_authorization_freezes_on_user_delete()
```

## 10.4 新增接口

```text
POST /api/users/{user_id}/authorizations/{auth_id}/suspend
POST /api/users/{user_id}/authorizations/{auth_id}/enable
```

## 10.5 验收标准

```text
1. active 授权停用后生成 frozen 记录。
2. 停用不返点。
3. 停用后 Authorization.status=suspended。
4. 启用后 FreezeRecord.status=released。
5. 启用不扣点。
6. 启用后 valid_until = now + remaining_hours。
7. Admin 删除用户时 frozen -> refunded。
```

# 十一、阶段 P8：设备接口与授权状态实时联动

## 11.1 阶段目标

停用授权后，旧 Access Token 在未过期前也不能继续心跳或拉设备。

## 11.2 改动范围

```text
app/services/device_service.py
app/routers/device.py
```

## 11.3 执行项

在以下路径校验 active Authorization：

```text
process_heartbeat()
get_device_list()
get_device_data()
```

校验条件：

```text
user_id
game_project_id
Authorization.status == active
valid_until 未过期
```

## 11.4 验收标准

```text
1. 停用授权后新登录失败。
2. 停用授权后 refresh 失败。
3. 停用授权后旧 AT 心跳失败。
4. 停用授权后旧 AT 拉设备失败。
5. 启用授权后可恢复登录和心跳。
```

# 十二、阶段 P9：账务中心冻结权益只读展示

## 12.1 阶段目标
