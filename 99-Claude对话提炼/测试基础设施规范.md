## 一、conftest 三层 patch 模式（D014）

### 为什么需要三层

FastAPI 应用中存在两类副作用来源：

1. **通过 DI 注入**：`Depends(get_main_db)`、`Depends(get_redis)` → 覆盖 `app.dependency_overrides` 即可
2. **绕过 DI 直接调用**：`from app.database import _main_session_factory`、`incr_rate_limit` 等 → DI 覆盖对这类**无效**，必须直接 patch 模块变量

### 三层 patch 结构

```python
# patch 1：FastAPI DI 覆盖
app.dependency_overrides[get_main_db] = _override_get_db
app.dependency_overrides[get_redis]   = _override_get_redis

# patch 2：模块级 session factory（_write_login_log 直接调用）
_app_database._main_session_factory   = session_factory   # app.database
_auth_service._main_session_factory   = session_factory   # app.services.auth_service

# patch 3：限流函数（测试共用 IP，会超出 10次/分钟）
async def _mock_rate_limit(*args, **kwargs):
    return True, 0   # 注意：必须返回 tuple，与路由层解包格式一致
_auth_router.incr_rate_limit = _mock_rate_limit
```

### 关键陷阱：from...import 的引用绑定

```python
# auth_service.py 中
from app.database import _main_session_factory   # 模块加载时绑定旧引用

# 测试中 patch app.database._main_session_factory 不影响 auth_service 内部变量
# 必须同时 patch：
import app.services.auth_service as _auth_service
_auth_service._main_session_factory = session_factory
```

### 识别需要 patch 的点

在被测服务代码中搜索：
- 顶层 `from app.xxx import yyy`（非 DI 注入，运行时持有旧引用）
- 直接调用 `session_factory()` 而非 `Depends(get_main_db)`

### 当前已知需要 patch 的点

| 模块 | 对象 | 触发场景 |
|------|------|---------|
| `app.services.auth_service` | `_main_session_factory` | 登录失败/成功均写 `login_log`，用独立 Session 防事务污染 |
| `app.routers.auth` | `incr_rate_limit` | 所有登录请求限流，测试中共用 IP 会超出 10次/分钟 |

## 二、游戏库访问规范（D015）

### 问题描述

在 pytest session scope fixture 中，使用 SQLAlchemy 连接游戏库（`hive_game_001`）时，
Windows ProactorEventLoop 与 asyncpg dialect 存在兼容问题，表现为：

```text
asyncpg.exceptions.InvalidPasswordError: password authentication failed for user "hive_user"
```

但实际上密码和权限均正确，用原生 `asyncpg.connect()` 直连完全正常。

### 规范

**测试中凡是需要直接操作游戏库的地方，一律使用原生 asyncpg，不用 SQLAlchemy engine。**

```python
# ✅ 正确：原生 asyncpg
conn = await asyncpg.connect(
    host="127.0.0.1", port=15432,
    user="hive_user", password="hive_dev_2026",
    database="hive_game_001", ssl=False,
)
await conn.execute("INSERT INTO ...")
await conn.close()

# ❌ 错误：SQLAlchemy engine（在 pytest session loop 中失败）
engine = create_async_engine(game_db_url, poolclass=NullPool, connect_args={"ssl": False})
async with engine.connect() as conn:
    await conn.execute(...)
```

### URL 派生方式（D013）

```python
from sqlalchemy.engine.url import make_url
from app.config import settings

def _game_db_dsn(code_name: str) -> dict:
    """从主库 URL 派生游戏库连接参数，避免双前缀 Bug。"""
    parsed = make_url(settings.DATABASE_MAIN_URL)
    return {
        "host": str(parsed.host),
        "port": int(parsed.port),
        "user": str(parsed.username),
        "password": str(parsed.password),
        "database": "hive_" + code_name,   # "hive_game_001"，不是 "hive_game_game_001"
        "ssl": False,
    }
```

## 三、测试数据命名规范

### 前缀隔离

测试专用数据加固定前缀，与开发数据区分：

```python
# 参数定义
"test_int_param", "test_bool_param", ...

# 用户名
f"param_test_{uuid.uuid4().hex[:8]}"
f"auth_test_{uuid.uuid4().hex[:8]}"
```

### Teardown 清理

fixture 的 teardown 按前缀清理，不影响开发数据：

```python
@pytest.fixture(scope="module")
async def seed_params():
    # setup：插入测试数据
    conn = await asyncpg.connect(**_game_db_dsn("game_001"))
    await conn.execute("INSERT INTO script_param_def ... ON CONFLICT DO NOTHING")
    await conn.close()

    yield

    # teardown：清理测试数据
    conn = await asyncpg.connect(**_game_db_dsn("game_001"))
    await conn.execute("DELETE FROM script_param_def WHERE param_key LIKE 'test_%'")
    await conn.close()
```

## 四、PostgreSQL 环境配置（D016，一次性）

### pg_hba.conf 认证方式

新版 PostgreSQL 默认密码存储为 `SCRAM-SHA-256`，但默认 TCP 规则可能是 `md5`，
两者不匹配导致认证失败。

**修复方法（WSL / Linux 环境）：**

```bash
sudo nano /etc/postgresql/16/main/pg_hba.conf
```

将：
```text
host    all    all    0.0.0.0/0    md5
```
改为：
```text
host    all    all    0.0.0.0/0    scram-sha-256
```

重载配置：
```bash
sudo -u postgres psql -p 15432 -c "SELECT pg_reload_conf();"
```

### 游戏库权限授予

`hive_user` 默认只有 `hive_platform` 的权限，每新增游戏库需执行一次：

```sql
GRANT CONNECT ON DATABASE hive_game_001 TO hive_user;
\c hive_game_001
GRANT USAGE ON SCHEMA public TO hive_user;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO hive_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO hive_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO hive_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO hive_user;
```

### 验证命令

```bash
# 验证 hive_user 能连接游戏库
psql -U hive_user -h 127.0.0.1 -p 15432 -d hive_game_001 -c "SELECT 1;"
```
