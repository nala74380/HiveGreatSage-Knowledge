# 2026-04-24 前端管理后台完整实现

## 对话目标

从零搭建 HiveGreatSage-Verify 的 Vue 3 管理后台前端，覆盖整体架构设计、目录结构、所有核心页面和组件，并在调试过程中修复若干后端问题。

---

## 一、关键结论

### 1. 前端放置位置
`HiveGreatSage-Verify/frontend/` 目录下，与后端同仓库。  
Vite 开发代理将 `/api` 和 `/admin/api` 转发到 `http://127.0.0.1:8000`（注意必须用 `127.0.0.1` 而非 `localhost`，Windows 11 DNS 解析问题）。

### 2. 双身份统一登录设计
管理员和代理共用同一个登录页，`stores/auth.js` 先尝试 Admin Token，失败再尝试 Agent Token，两种身份不需要切换入口。

**Admin LoginResponse 字段：** `{ access_token, token_type, expires_in, admin_id, username }`  
**Agent LoginResponse 字段：** `{ access_token, token_type, expires_in, agent_id, username, level }`

Admin/Agent Token **均无 refresh_token**，过期直接清除状态跳转 `/login`，不做静默刷新。

### 3. 视图隔离靠「数据范围」不靠「路由隔离」
同一组件页面（如 `UserList.vue`），Admin 调 `/admin/api` 或带管理权限的接口拿全量数据，Agent 调相同接口但后端 `_resolve_caller` 自动按 Token 类型过滤数据范围，前端无需维护两套页面。

### 4. `_skipAuthRedirect` 拦截器选项
`api/http.js` 新增 `_skipAuthRedirect: true` 配置项。  
背景：`/api/device/list` 只接受 User Token，Admin/Agent Token 调用会收到 401（权限不足，**非** Token 失效），此时不应强制跳登录页，而应静默处理。  
凡涉及「有 Token 但无权限调用的端点」时，请求配置加此选项。

### 5. 设备监控 Phase 2 待办（T021）
管理后台设备监控页面当前因权限问题显示占位提示。  
**Phase 2 待实现：** `GET /admin/api/devices/` 接受 Admin/Agent Token，替代当前 `notSupported` 占位逻辑。  
已记录到 [[待决策清单]] T021。

### 6. 用户管理功能扩展（本次对话完成）
- **授权数量 / 已激活数量**：后端 `list_users` 新增 3 条聚合子查询，一次性返回 `authorization_count`、`active_authorization_count`、`device_binding_count`，避免 N+1。
- **到期时间显示**：列表展示剩余天数彩色 Tag（红≤7天/橙≤30天/绿正常），新建/编辑支持快捷按钮（一周/一个月/三个月）。
- **编辑续费设计**：编辑框分三区——只读显示当前到期时间 + 续费增量（从当前到期日续起，实时预览新日期）+ 直接指定新日期。
- **`max_devices` 独立字段**：设备绑定上限不再由级别自动决定，改为每用户独立设置（`INTEGER NOT NULL DEFAULT 500`），新建/编辑均可修改，快捷按钮：100/500/1000/+100/+500/无限制。

## 三、后端同步修改

`migrations/versions/0003_add_user_max_devices.py`  
给 `user` 表新增 `max_devices INTEGER NOT NULL DEFAULT 500` 字段。

**迁移命令：**

### 修改文件
| 文件 | 改动摘要 |
|------|---------|

| `app/schemas/user.py` | `UserCreateRequest` 加 `max_devices`（默认500）；`UserUpdateRequest` 加 `max_devices`；`UserResponse` 加 `max_devices` + `authorization_count` + `active_authorization_count` + `device_binding_count` |
| `app/services/user_service.py` | `list_users` 加 3 条聚合子查询；`create_user` / `update_user` / `_user_to_response` 支持 `max_devices`；移除错误的 `func.cast(... db.bind.dialect.INTEGER ...)` 写法 |

## 四、存疑 / 已验证的技术点

| 问题 | 结论 |
|------|------|
| Windows 11 `localhost` DNS 失败 | 改用 `127.0.0.1`，`vite.config.js` 的 `server.host` 和 `proxy.target` 均使用 `127.0.0.1` |
| Admin Token 调 `/api/device/list` 返回 401 | 该端点只接受 User Token，非 Token 失效，加 `_skipAuthRedirect: true` 静默处理，Phase 2 新增 `/admin/api/devices/` 解决 |
| `func.cast(... db.bind.dialect.INTEGER ...)` 报错 | PostgreSQL asyncpg 方言对象无 `INTEGER` 属性，改为两条独立的 COUNT 查询即可 |
| Admin/Agent Token 是否需要 Refresh Token | 不需要，后端设计上均无 refresh_token，过期直接跳登录 |

## 五、启动方式

```bash
# 前端
cd HiveGreatSage-Verify/frontend
npm install
npm run dev
# 访问 http://127.0.0.1:5173

# 后端（先执行迁移再启动）
cd HiveGreatSage-Verify
alembic upgrade head
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

# 初始账号（需先运行 scripts/init_data.py）
# 管理员：admin / Admin@2026!
```
